    </button></h1><p>Der Filter <code translate="no">synonym</code> schreibt Token entsprechend einem Synonymwörterbuch um, so dass verwandte Begriffe bei der Suche übereinstimmen. Er unterstützt zwei Betriebsarten und zwei Arten der Bereitstellung des Wörterbuchs:</p>
<ul>
<li><p><strong>Betriebsarten</strong> - im Modus <code translate="no">expand</code> wird das ursprüngliche Token beibehalten und es werden zusätzliche Synonyme daneben ausgegeben; im Normalisierungsmodus (<code translate="no">expand: false</code>) werden Token in eine kanonische Form umgeschrieben.</p></li>
<li><p><strong>Wörterbuchquellen</strong> - kleine Wörterbücher können über das Array <code translate="no">synonyms</code> in die Filterkonfiguration eingefügt werden; große Wörterbücher sollten als <a href="/docs/de/manage-file-resources.md">Dateiressource</a> gespeichert und über <code translate="no">synonyms_file</code> referenziert werden.</p></li>
</ul>

        ></path>
      </svg>
    </button></h3><pre><code translate="no" class="language-plaintext">fast, quick =&gt; speedy
<button class="copy-code-btn"></button></code></pre>
<p>Die Token auf der linken Seite (<code translate="no">fast</code>, <code translate="no">quick</code>) werden auf die Token auf der rechten Seite (<code translate="no">speedy</code>) umgeschrieben. Mehrere Ziele sind erlaubt:</p>
<pre><code translate="no" class="language-plaintext">small, little =&gt; tiny, compact
<button class="copy-code-btn"></button></code></pre>
<p>Bei <code translate="no">expand: true</code> werden die ursprünglichen Zeichen neben den Zielen beibehalten:</p>
<ul>
<li><p>Eingabe <code translate="no">fast</code> mit <code translate="no">expand: true</code> → <code translate="no">fast</code>, <code translate="no">speedy</code></p></li>
<li><p>Eingabe <code translate="no">fast</code> mit <code translate="no">expand: false</code> → <code translate="no">speedy</code></p></li>
</ul>

<p>Alle aufgeführten Zeichen werden als gleichwertig betrachtet:</p>
<ul>
<li><p>Mit <code translate="no">expand: true</code> gibt jedes Vorkommen eines beliebigen Tokens in der Gruppe jedes Token in der Gruppe aus. Eingabe <code translate="no">happy</code> → <code translate="no">happy</code>, <code translate="no">joyful</code>, <code translate="no">cheerful</code>.</p></li>
<li><p>Mit <code translate="no">expand: false</code> wird jedes Vorkommen auf das erste Token in der Gruppe umgeschrieben. Eingabe <code translate="no">joyful</code> → <code translate="no">happy</code>; die Eingabe <code translate="no">happy</code> ist bereits das erste Token und bleibt unverändert.</p></li>
</ul>

Geben Sie <code translate="no">&quot;type&quot;: &quot;synonym&quot;</code> zusammen mit mindestens einem von <code translate="no">synonyms</code> (inline) oder <code translate="no">synonyms_file</code> (extern) sowie einem <code translate="no">expand</code> Flag an.</p>

<p>Der Filter <code translate="no">synonym</code> akzeptiert die folgenden Parameter.</p>
<table>
   <tr>
     <th><p><strong>Parameter</strong></p></th>
     <th><p><strong>Beschreibung</strong></p></th>
     <th><p><strong>Voreinstellung</strong></p></th>
   </tr>
   <tr>
     <td><p><code translate="no">synonyms</code></p></td>
     <td><p>Ein Inline-Array von Regelzeichenketten. Jede Zeichenkette verwendet das oben beschriebene Wörterbuchformat. Geeignet für kleine Wörterbücher (bis zu ein paar Dutzend Regeln).</p></td>
     <td><p>-</p></td>
   </tr>
   <tr>
     <td><p><code translate="no">synonyms_file</code></p></td>
     <td><p>Ein Verweis auf eine <a href="/docs/de/manage-file-resources.md">Dateiressource</a>, die Synonymregeln speichert, eine pro Zeile. Für größere Wörterbücher zu verwenden. Siehe <a href="/docs/de/synonym-filter.md#External-dictionary-file">Externe Wörterbuchdatei</a> unten.</p></td>
     <td><p>-</p></td>
   </tr>
   <tr>
     <td><p><code translate="no">expand</code></p></td>
     <td><p>Ein boolesches Flag, das steuert, wie die Regeln angewendet werden. true bewahrt das ursprüngliche Token und gibt daneben Synonyme aus; false schreibt Token in ihre kanonische Form um (die rechte Seite einer Abbildung oder das erste Token einer Äquivalenzgruppe).</p></td>
     <td><p>false</p></td>
   </tr>
</table>
<p>Sie können <code translate="no">synonyms</code>, <code translate="no">synonyms_file</code>, oder beides angeben. Wenn beides vorhanden ist, führt der Filter die beiden Quellen zusammen. Der Filter arbeitet mit Token, die vom Tokenizer erzeugt werden; er muss daher mit einem Tokenizer wie dem <a href="/docs/de/standard-tokenizer.md">Standard-Tokenizer</a> kombiniert werden.</p>
