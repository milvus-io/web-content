    </button></h1><p>Die <strong>BM25-Funktion</strong> ermöglicht eine <a href="/docs/de/full-text-search.md">Volltextsuche</a>, indem sie Rohtext in <strong>spärliche Vektoren</strong> umwandelt und Dokumente auf der Grundlage lexikalischer Relevanz bewertet. Sie wendet termbasiertes Matching und frequenzabhängige Gewichtung an, um eine effiziente Suche nach Textdokumenten zu unterstützen, die eng mit den Suchbegriffen übereinstimmen.</p>
<p>Als lokale Textfunktion läuft die BM25-Funktion innerhalb von Milvus und erfordert keine Modellinferenz oder externe Integrationen. Sie bietet einen deterministischen und transparenten Abrufmechanismus für textbasierte Suchszenarien.</p>

    </button></h2><p>Der <a href="https://en.wikipedia.org/wiki/Okapi_BM25">BM25-Algorithmus</a> ist ein begriffsbasierter Relevanzbewertungsalgorithmus, der in der Volltextsuche weit verbreitet ist. In Milvus ist BM25 als Sparse-Retrieval-Pipeline implementiert, die Text in begriffsgewichtete Repräsentationen umwandelt und die <em>Top-K-Dokumente</em> mithilfe verteilter Sparse-Indizes abruft.</p>
<p>Der gesamte Arbeitsablauf besteht aus zwei symmetrischen Pfaden: <strong>Dokumenteneingabe</strong> und <strong>Textverarbeitung</strong>, die dieselbe Textanalyselogik nutzen.</p>

      </svg>
    </button></h3><p>Wenn ein Dokument eingefügt wird, wird sein Rohtext zunächst von einem <strong><a href="/docs/de/analyzer-overview.md">Analysator</a></strong> verarbeitet, der den Text in einzelne Begriffe aufschlüsselt.</p>
<p>Zum Beispiel kann das Dokument:</p>
<pre><code translate="no" class="language-plaintext">&quot;We are loving Milvus!&quot;
<button class="copy-code-btn"></button></code></pre>
<p>kann in die folgenden Begriffe zerlegt werden:</p>
<pre><code translate="no" class="language-plaintext">[&quot;we&quot;, &quot;love&quot;, &quot;milvus&quot;]
<button class="copy-code-btn"></button></code></pre>
<p>Jedes Dokument wird dann als Termfrequenzdarstellung (TF) dargestellt, die aufzeichnet, wie oft jeder Begriff in dem Dokument vorkommt. Ein Beispiel:</p>
<pre><code translate="no" class="language-plaintext">{
  &quot;we&quot;: 1,
  &quot;love&quot;: 1,
  &quot;milvus&quot;: 1
}
<button class="copy-code-btn"></button></code></pre>
<p>Gleichzeitig aktualisiert Milvus Statistiken auf Korpusebene, darunter:</p>
<ul>
<li><p>die Dokumenthäufigkeit (DF) eines jeden Begriffs</p></li>
<li><p>die durchschnittliche Dokumentlänge</p></li>
<li><p>Buchungslisten, die jeden Begriff den Dokumenten zuordnen, die ihn enthalten</p></li>
</ul>
<p>Die TF-Darstellung des Dokuments wird in <strong>Sparse Embeddings</strong> eingefügt, in denen die Termpostings für eine skalierbare Abfrage auf die Knoten verteilt werden.</p>

    </button></h3><p>Wenn eine textbasierte Abfrage gestellt wird, wird sie von <strong>demselben Analysator</strong> verarbeitet, der auch bei der <a href="/docs/de/bm25-function.md#Document-ingestion-From-text-to-sparse-representation">Aufnahme von Dokumenten</a> verwendet wird, um eine konsistente Termsegmentierung sicherzustellen.</p>
<p>Ein Beispiel: Die Abfrage:</p>
<pre><code translate="no" class="language-plaintext">&quot;who loves Milvus?&quot;
<button class="copy-code-btn"></button></code></pre>
<p>kann analysiert werden in:</p>
<pre><code translate="no" class="language-plaintext">[&quot;who&quot;, &quot;love&quot;, &quot;milvus&quot;]
<button class="copy-code-btn"></button></code></pre>
<p>Für jeden Abfrageterminus ermittelt Milvus seine <a href="https://en.wikipedia.org/wiki/Tf%E2%80%93idf">inverse Dokumentenhäufigkeit</a> (IDF) aus den Korpusstatistiken. Die IDF spiegelt wider, wie informativ ein Begriff im gesamten Datensatz ist: Seltene Begriffe erhalten eine höhere Gewichtung, während häufige Begriffe eine geringere Gewichtung erhalten.</p>
<p>Daraus ergibt sich eine Reihe von IDF-gewichteten Suchbegriffen, wie z. B.:</p>
<pre><code translate="no" class="language-plaintext">{
  &quot;who&quot;: 0.1,
  &quot;love&quot;: 0.5,
  &quot;milvus&quot;: 1.2
}
<button class="copy-code-btn"></button></code></pre>
<h3 id="BM25-scoring-and-top-K-retrieval" class="common-anchor-header">BM25-Bewertung und Top-K-Retrieval<button data-href="#BM25-scoring-and-top-K-retrieval" class="anchor-icon" translate="no">
      <svg translate="no"
        aria-hidden="true"

      >
        <path

          d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"
        ></path>
      </svg>
    </button></h3><p>BM25 ordnet Dokumente ein, indem es eine Relevanzbewertung auf der Grundlage der übereinstimmenden Suchbegriffe berechnet. Die Bewertung erfolgt auf <strong>Begriffsebene</strong> und wird auf <strong>Dokumentenebene</strong> aggregiert.</p>
<p><strong>Bewertung auf Termebene</strong></p>
<p>Für jeden Suchbegriff, der in einem Dokument vorkommt, berechnet BM25 eine Bewertung auf Termebene:</p>
<pre><code translate="no" class="language-plaintext">term_score =
  IDF(term) ×
  TF_boost(term, document, k1) ×
  length_normalization(document, b)
<button class="copy-code-btn"></button></code></pre>
<p>Wobei:</p>
<ul>
<li><p><strong>IDF(term)</strong> gibt an, wie selten der Begriff in der Sammlung ist</p></li>
<li><p><strong>TF_boost(..., k1)</strong> steigt mit der Termhäufigkeit, sättigt aber mit zunehmender Häufigkeit</p></li>
<li><p><strong>length_normalization(..., b)</strong> passt die Punktzahl auf der Grundlage der Dokumentenlänge an</p></li>
</ul>
<p><strong>Scoring auf Dokumentenebene und Top-K Retrieval</strong></p>
<p>Die endgültige Dokumentenbewertung ist die Summe der Bewertungen auf Termebene für alle übereinstimmenden Suchbegriffe:</p>
<pre><code translate="no" class="language-plaintext">document_score =
  sum of term_score over all matched query terms
<button class="copy-code-btn"></button></code></pre>
<p>Die Dokumente werden nach ihren endgültigen Punktzahlen geordnet, und die Top-K-Dokumente mit den höchsten Punktzahlen werden zurückgegeben.</p>

    </button></h2><p>Bevor Sie die BM25-Funktion verwenden, planen Sie Ihr Sammlungsschema, um sicherzustellen, dass es die lexikalische Volltextsuche unterstützt:</p>
<ul>
<li><p><strong>Ein Textfeld für Rohinhalte</strong></p>
<p>Ihre Sammlung muss ein <code translate="no">VARCHAR</code> Feld zur Speicherung von Rohtext enthalten. Dieses Feld ist die Quelle des Textes, der für die Volltextsuche verarbeitet werden soll.</p></li>
<li><p><strong>Ein Analysator für das Textfeld</strong></p>
<p>Für das Textfeld muss ein Analyzer aktiviert sein. Der Analyzer definiert, wie der Text tokenisiert und normalisiert wird, bevor die lexikalische Relevanz von der BM25-Funktion berechnet wird.</p>
<p>Standardmäßig bietet Milvus einen eingebauten Analyzer, der Text auf der Grundlage von Leerzeichen und Interpunktion in Token umwandelt. Wenn Ihre Anwendung ein benutzerdefiniertes Tokenisierungs- oder Normalisierungsverhalten erfordert, können Sie einen benutzerdefinierten Analyzer definieren. Weitere Informationen finden Sie unter <a href="/docs/de/choose-the-right-analyzer-for-your-use-case.md">Wählen Sie den richtigen Analyzer für Ihren Anwendungsfall</a>.</p></li>
<li><p><strong>Ein spärlicher Vektor für die BM25-Ausgabe</strong></p>
<p>Ihre Sammlung muss ein Feld <code translate="no">SPARSE_FLOAT_VECTOR</code> enthalten, um die von der BM25-Funktion erzeugten Sparse-Darstellungen zu speichern. Dieses Feld wird für die Indizierung und den Abruf bei der Volltextsuche verwendet.</p></li>
</ul>
<p>Nachdem diese Überlegungen auf Schemaebene geklärt sind, fahren Sie mit der Erstellung der Sammlung und der Verwendung der BM25-Funktion fort.</p>
