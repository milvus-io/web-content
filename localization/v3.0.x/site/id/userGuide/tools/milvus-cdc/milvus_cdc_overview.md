    </button></h1><p>Milvus CDC (Change Data Capture) mereplikasi perubahan data dari satu cluster Milvus ke cluster lainnya. Anda bisa menggunakan CDC untuk membangun topologi pemulihan bencana siaga-primer untuk Milvus.</p>
<p>Dalam topologi primary-standby, satu cluster bertindak sebagai primary dan menerima penulisan. Satu atau lebih cluster siaga secara terus menerus menerima perubahan dari cluster utama dan dapat melayani lalu lintas baca. Ketika cluster utama tidak tersedia atau membutuhkan pemeliharaan, Anda bisa mengalihkan lalu lintas layanan ke cluster siaga.</p>

    </button></h2><p>Topologi yang umum terdiri dari:</p>
<ul>
<li><strong>Cluster primer</strong>: Cluster sumber untuk replikasi. Klaster ini menerima pembacaan dan penulisan.</li>
<li><strong>Klaster siaga</strong>: Sebuah cluster target untuk replikasi. Cluster ini menerima perubahan dari cluster primer dan hanya dapat dibaca saat masih dalam keadaan siaga.</li>
<li><strong>Simpul CDC</strong>: Komponen Milvus yang meneruskan perubahan WAL dari cluster utama saat ini ke cluster siaga. Menerapkan CDC pada setiap cluster yang mungkin menjadi primer setelah peralihan atau peralihan.</li>
<li><strong>Topologi replikasi</strong>: Hubungan sumber-ke-target yang dikonfigurasi, seperti cluster-a -&gt; cluster-b. Berikut ini adalah ilustrasi dari topologi tersebut. <span class="img-wrapper"> <img translate="no" src="/docs/v3.0.x/assets/cdc-overview.png" alt="CDC workflow" class="doc-image" id="cdc-workflow" /><span>Alur kerja CDC</span> </span></li>
</ul>

<p>Jeda CDC memengaruhi kedua opsi pemulihan:</p>
<ul>
<li>Selama peralihan, jeda CDC yang lebih rendah biasanya berarti operasi selesai lebih cepat.</li>
<li>Selama failover, jeda CDC menunjukkan jendela data yang mungkin hilang jika primary asli tidak tersedia.</li>
</ul>
<p>Anda harus memantau jeda CDC secara terus menerus dan menjaganya serendah mungkin. Halaman <a href="/docs/id/set_up_cdc_replication.md">Siapkan Replikasi CDC</a> menyertakan contoh PromQL untuk memperkirakan jeda CDC.</p>

<ul>
<li>Hanya mendukung topologi <strong>primer tunggal</strong>.</li>
<li><strong>Tidak</strong> mendukung penulisan aktif-aktif atau multi-primer.</li>
<li>Klaster siaga dapat melayani lalu lintas baca, tetapi mereka menolak penulisan langsung saat mereka tetap siaga.</li>
<li>Failover dapat kehilangan data yang telah ditulis ke primary lama tetapi belum direplikasi ke standby.</li>
<li><code translate="no">pchannels</code> yang dikonfigurasi harus sesuai dengan tata letak saluran yang sebenarnya dari setiap cluster.</li>
</ul>
