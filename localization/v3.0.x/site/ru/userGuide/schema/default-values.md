    </button></h1><p>Milvus позволяет устанавливать значения по умолчанию для скалярных полей (за исключением первичного поля). Если для поля задано значение по умолчанию, Milvus автоматически применяет это значение, если при вставке не было предоставлено никаких данных.</p>
<p>Значения по умолчанию упрощают миграцию данных из других систем баз данных в Milvus, сохраняя существующие настройки значений по умолчанию. Вы также можете использовать значения по умолчанию для полей, значения которых могут быть неопределенными на момент вставки.</p>

        ></path>
      </svg>
    </button></h2><ul>
<li><p>Только скалярные поля поддерживают значения по умолчанию. Первичное поле и векторные поля не могут иметь значения по умолчанию.</p></li>
<li><p><code translate="no">JSON</code> и поля <code translate="no">ARRAY</code> не поддерживают значения по умолчанию.</p></li>
<li><p>Значения по умолчанию можно настроить только во время создания коллекции и не могут быть изменены впоследствии.</p></li>
</ul>

<p><strong>Основные выводы:</strong></p>
<ul>
<li><p>Если поле имеет не NULL-значение по умолчанию, это значение используется независимо от того, включен ли <code translate="no">nullable</code>.</p></li>
<li><p>Если включен <code translate="no">nullable=True</code>, но не установлено значение по умолчанию, в поле сохраняется NULL.</p></li>
<li><p>Если <code translate="no">nullable=False</code> и значение по умолчанию не установлено, вставка завершится с ошибкой.</p></li>
<li><p>Установка значения по умолчанию NULL для поля, не являющегося нулевым, недействительна и приводит к ошибке.</p></li>
</ul>
