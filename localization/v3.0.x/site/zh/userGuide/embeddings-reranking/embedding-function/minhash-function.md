    </button></h1><p><strong>MinHash 函数</strong>可将原始文本转换为<strong>二进制向量</strong>，以接近文档之间的<a href="https://en.wikipedia.org/wiki/Jaccard_index">Jaccard 相似性</a>。它应用文本切分和多个哈希函数来生成固定长度的签名向量，从而实现快速的近似重复检测和大规模的文档重复删除。</p>
<p>作为一个内置函数，MinHash 在 Milvus 内部运行，不需要外部模型推断或预处理。你只需插入原始文本，Milvus 就会自动生成 MinHash 签名向量。</p>

<p><summary>展开查看其工作原理</summary></p>
<p><a href="https://en.wikipedia.org/wiki/MinHash">MinHash</a>是一种对位置敏感的散列技术，可估算集合间的<a href="https://en.wikipedia.org/wiki/Jaccard_index">Jaccard 相似性</a>。在 Milvus 中，MinHash 功能遵循以下流程：您提供原始文本作为输入，Milvus 生成二进制向量作为输出--内部处理所有中间步骤。</p>
<p>整个工作流程包括一个<strong>共享的文本处理管道</strong>，文档摄取和查询处理都会用到，然后是存储和检索的特定阶段操作符。</p>

        ></path>
      </svg>
    </button></h3><p>文档摄取和查询处理都通过相同的四阶段转换来处理原始文本：</p>
<ol>
<li><p><strong>文本分析</strong>：文本由<a href="/docs/zh/analyzer-overview.md">分析器</a>处理（当<code translate="no">token_level</code> 是<code translate="no">&quot;word&quot;</code> 时）或直接使用（当<code translate="no">token_level</code> 是<code translate="no">&quot;char&quot;</code> 时）。词级标记化应用在输入字段上配置的分析器，将文本分割成术语--例如，<code translate="no">&quot;milvus is vector db&quot;</code> 变成<code translate="no">[&quot;milvus&quot;, &quot;is&quot;, &quot;vector&quot;, &quot;db&quot;]</code> 。</p></li>
<li><p><strong>分层</strong>：词块被分割成大小为<code translate="no">shingle_size</code> 的重叠 n-gram（shingles）。例如，在单词级别使用 3-grams 时，令牌<code translate="no">[&quot;information&quot;, &quot;retrieval&quot;, &quot;is&quot;, &quot;a&quot;, &quot;field&quot;]</code> 就会变成<code translate="no">[&quot;information retrieval is&quot;, &quot;retrieval is a&quot;, &quot;is a field&quot;]</code> 这样的词块。</p></li>
<li><p><strong>MinHash 签名生成</strong>：将多个哈希函数（H1、H2、...、Hn，其中 n =<code translate="no">num_hashes</code> ）应用于楔形字符集。对于每个哈希函数，都会在所有木片中选择最小哈希值。这些最小值的 Collections 构成了 MinHash 签名--一个固定长度的表示，近似于原始文档的 Jaccard 相似度。</p></li>
<li><p><strong>二进制向量编码</strong>：每个签名值都是一个 32 位的哈希值，完整签名被打包到维度为<code translate="no">BINARY_VECTOR</code> 的<code translate="no">32 * num_hashes</code> 中。</p></li>
</ol>

    </button></h3><p>在搜索过程中，查询文本通过相同的共享管道产生二进制向量。该向量用于在<code translate="no">MINHASH_LSH</code> 索引中执行 LSH 查找，从而快速确定可能相似的候选配对。然后根据估计的 Jaccard 相似度对候选结果进行排序，并返回前 K 个结果。</p>
<p>由于两个路径共享相同的转换逻辑，内容高度重叠的两个文档会产生相似的 MinHash 签名。因此，即使文档在词序、格式或细微措辞上存在差异，该功能也能有效查找近似重复的文档。</p>

</ul>

<p><strong>配置选项</strong></p>
<p>MinHash 函数的<code translate="no">params</code> 字典接受以下参数。所有参数名均<strong>不区分大小写</strong>。</p>
<table>
   <tr>
     <th><p><strong>参数</strong></p></th>
     <th><p><strong>类型</strong></p></th>
     <th><p><strong>默认值</strong></p></th>
     <th><p><strong>说明</strong></p></th>
   </tr>
   <tr>
     <td><p><code translate="no">num_hashes</code></p></td>
     <td><p>int</p></td>
     <td><p>从<code translate="no">dim / 32</code></p></td>
     <td><p>用于生成签名的哈希函数个数。输出二进制向量维数等于<code translate="no">32 &ast; num_hashes</code> 。数值越大，相似性估计的方差越小，但计算量会增加。建议：<code translate="no">256</code> （dim = 8192）。</p></td>
   </tr>
   <tr>
     <td><p><code translate="no">shingle_size</code></p></td>
     <td><p>输入</p></td>
     <td><p><code translate="no">3</code></p></td>
     <td><p>用于分层的 N-gram 大小。词级：通常为 1-3。字符级：一般为 2-6。</p></td>
   </tr>
   <tr>
     <td><p><code translate="no">hash_function</code></p></td>
     <td><p>字符串</p></td>
     <td><p><code translate="no">"xxhash"</code></p></td>
     <td><p>要使用的哈希函数。选项： </p><ul><li><p><code translate="no">"xxhash"</code> 快</p></li><li><p><code translate="no">"sha1"</code> (较慢，抗碰撞能力较强）。</p></li></ul></td>
   </tr>
   <tr>
     <td><p><code translate="no">token_level</code></p></td>
     <td><p>str</p></td>
     <td><p><code translate="no">"word"</code></p></td>
     <td><p>令牌化级别。选项：</p><ul><li><p><code translate="no">"word"</code>：使用字段分析器进行标记化，然后应用 n-gram shingling。</p></li><li><p><code translate="no">"char"</code> /<code translate="no">"character"</code> ：直接在原始字符上应用 n-gram shingling（不使用分析器）。</p><p>单词级提供了更强的语义和更高的效率，但依赖于特定语言的标记化。字符级与语言无关，但能产生语义较弱的高维分词。</p></li></ul></td>
   </tr>
   <tr>
     <td><p><code translate="no">seed</code></p></td>
     <td><p>随机种子</p></td>
     <td><p><code translate="no">1234</code></p></td>
     <td><p>用于 MinHash 函数初始化的随机种子。</p></td>
   </tr>
</table>
