    </button></h2><div class="alert note">
<p>For Arabic text, the built-in <a href="/docs/arabic-analyzer.md"><code translate="no">arabic</code></a> analyzer already includes the <code translate="no">decimaldigit</code> filter. Use <code translate="no">decimaldigit</code> directly when you need digit normalization in a custom analyzer pipeline.</p>
</div>
<p>To use the <code translate="no">decimaldigit</code> filter in a custom analyzer, add it to the <code translate="no">filter</code> section in <code translate="no">analyzer_params</code>:</p>
<pre><code translate="no" class="language-python">analyzer_params = {
    <span class="hljs-string">&quot;tokenizer&quot;</span>: <span class="hljs-string">&quot;standard&quot;</span>,
    <span class="hljs-string">&quot;filter&quot;</span>: [<span class="hljs-string">&quot;decimaldigit&quot;</span>],
}
<button class="copy-code-btn"></button></code></pre>
<p>The <code translate="no">decimaldigit</code> filter has no configurable parameters.</p>
<p>The filter converts Unicode decimal digits, including Arabic-Indic, Thai, Devanagari, Bengali, and fullwidth digits, to ASCII digits. It operates on tokens generated by the tokenizer. The configuration above is intentionally a custom analyzer example and does not include the complete Arabic processing pipeline.</p>
