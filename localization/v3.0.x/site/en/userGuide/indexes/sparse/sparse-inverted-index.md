    </button></h1><p>The <code translate="no">SPARSE_INVERTED_INDEX</code> index is an index type used by Milvus to efficiently store and search sparse vectors. It builds an inverted structure from the non-zero dimensions in sparse vectors. You can use this index for BM25 full text search and for sparse embedding search based on inner product.</p>
<p>For more information about sparse vector fields, metric types, and full text search, refer to <a href="/docs/sparse_vector.md">Sparse Vector</a>, <a href="/docs/metric.md">Metric Types</a>, and <a href="/docs/full-text-search.md">Full Text Search</a>.</p>

<p>For BM25 full text search, build the index on the sparse vector field generated by a BM25 function. Set <code translate="no">metric_type</code> to <code translate="no">BM25</code>.</p>

<span class="hljs-comment"># Prepare index building params</span>

index_params.add_index(
    field_name=<span class="hljs-string">&quot;sparse&quot;</span>, <span class="hljs-comment"># Name of the sparse vector field to index</span>
    index_type=<span class="hljs-string">&quot;SPARSE_INVERTED_INDEX&quot;</span>, <span class="hljs-comment"># Type of the index to create</span>
    index_name=<span class="hljs-string">&quot;sparse_bm25_index&quot;</span>, <span class="hljs-comment"># Name of the index to create</span>

)

)

<p>In the preceding configurations:</p>
<ul>
<li><p><code translate="no">index_type</code>: The type of index to build. Set this value to <code translate="no">SPARSE_INVERTED_INDEX</code>.</p></li>
<li><p><code translate="no">metric_type</code>: The metric used to calculate similarity between sparse vectors. Valid Values:</p>
<ul>
<li><p><code translate="no">BM25</code>: Uses BM25 relevance scoring for full text search.</p></li>
<li><p><code translate="no">IP</code> (Inner Product): Measures sparse vector similarity using dot product.</p></li>
</ul>
<p>For details, refer to <a href="/docs/metric.md">Metric Types</a> and <a href="/docs/full-text-search.md">Full Text Search</a>.</p></li>
<li><p><code translate="no">params.inverted_index_algo</code>: The algorithm used for building and querying the index. Valid values:</p>
<ul>
<li><p><code translate="no">&quot;DAAT_MAXSCORE&quot;</code>: Document-at-a-Time MaxScore query processing. This is the default for <code translate="no">BM25</code>. For background, refer to <a href="https://dl.acm.org/doi/10.1016/0306-4573%2895%2900020-H">Query Evaluation: Strategies and Optimizations</a>.</p></li>
<li><p><code translate="no">&quot;DAAT_WAND&quot;</code>: Document-at-a-Time WAND query processing. This algorithm is suitable for smaller topK values or shorter queries. For background, refer to <a href="https://dl.acm.org/doi/10.1145/956863.956944">Efficient Query Evaluation using a Two-Level Retrieval Process</a>.</p></li>
<li><p><code translate="no">&quot;TAAT_NAIVE&quot;</code>: Basic Term-at-a-Time query processing. Use this option as a baseline or when you need scoring to adapt dynamically to global collection statistics such as average document length.</p></li>
<li><p><code translate="no">&quot;BLOCK_MAX_MAXSCORE&quot;</code>: MaxScore query processing with block-level max-score metadata. For background, refer to <a href="https://dl.acm.org/doi/10.1145/2009916.2010048">Faster Top-k Document Retrieval Using Block-Max Indexes</a>.</p></li>
<li><p><code translate="no">&quot;BLOCK_MAX_WAND&quot;</code>: WAND query processing with block-level max-score metadata. For background, refer to <a href="https://dl.acm.org/doi/10.1145/2009916.2010048">Faster Top-k Document Retrieval Using Block-Max Indexes</a>.</p></li>
<li><p><code translate="no">&quot;SINDI&quot;</code>: A sparse inverted index based on fixed document-id windows, with SIMD acceleration for search. This is the default for <code translate="no">IP</code>. For details, refer to the <a href="https://arxiv.org/abs/2509.08395">SINDI paper</a>.</p></li>
</ul>
<p>If you do not specify <code translate="no">inverted_index_algo</code>, Milvus selects the default algorithm based on <code translate="no">metric_type</code>: <code translate="no">DAAT_MAXSCORE</code> for <code translate="no">BM25</code>, and <code translate="no">SINDI</code> for <code translate="no">IP</code>.</p>
<p>To learn more building parameters available for the <code translate="no">SPARSE_INVERTED_INDEX</code> index, refer to <a href="/docs/sparse-inverted-index.md#Index-building-params">Index building params</a>.</p></li>
</ul>
<p>Once the index parameters are configured, you can create the index by using the <code translate="no">create_index()</code> method directly or passing the index params in the <code translate="no">create_collection</code> method. For details, refer to <a href="/docs/create-collection.md">Create Collection</a>.</p>
