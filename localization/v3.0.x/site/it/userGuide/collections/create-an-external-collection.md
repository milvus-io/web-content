Agisce come un livello di interrogazione sui data lake, mantenendo la compatibilità con le interfacce di interrogazione di Milvus.</p>

    </button></h2><p>In una tipica pipeline di dati AI, gli utenti potrebbero aver già memorizzato i loro dati in Parquet o in altri formati sul loro sistema di archiviazione, come AWS S3. Per far sì che Milvus utilizzi questi dati archiviati esternamente, gli utenti di solito devono importarli nello storage di Milvus utilizzando pipeline ETL (Extract-Transform-Load).</p>
<p>Questo flusso di lavoro "porta i tuoi dati a Milvus" crea dati ridondanti, difficili da sincronizzare, e aumenta l'onere di manutenzione tecnica per garantire la coerenza dei dati.</p>
<p>
  
   <span class="img-wrapper"> <img translate="no" src="/docs/v3.0.x/assets/yqxwwpq3vheya4b8398cwopnnyn.png" alt="Yqxwwpq3vheya4b8398cwopnnyn" class="doc-image" id="yqxwwpq3vheya4b8398cwopnnyn" />
   </span> <span class="img-wrapper"> <span>Yqxwwpq3vheya4b8398cwopnnyn</span> </span></p>
<p>Per risolvere questi problemi, Milvus offre collezioni esterne che consentono di accedere ai dati archiviati esternamente da Milvus senza preoccuparsi della sincronizzazione dei dati e delle pipeline ETL.</p>
<p>
  
   <span class="img-wrapper"> <img translate="no" src="/docs/v3.0.x/assets/q6f4wtcd2h3pnkbnmxncw3urn3f.png" alt="Q6f4wtcd2h3pnkbnmxncw3urn3f" class="doc-image" id="q6f4wtcd2h3pnkbnmxncw3urn3f" />
   </span> <span class="img-wrapper"> <span>Q6f4wtcd2h3pnkbnmxncw3urn3f</span> </span></p>
<p>Una volta creata, una raccolta esterna può accedere direttamente ai vostri dati e conservarli nello stesso luogo in cui li avete memorizzati. In background, Milvus crea file manifest per registrare le mappature tra i metadati Milvus e le righe dei file di dati esterni. Dopo che i file manifest sono pronti, è possibile creare indici nella collezione esterna come in qualsiasi altra collezione gestita.</p>
<p>Quando i dati cambiano, l'attivazione manuale di un aggiornamento al secondo aggiorna i metadati, mantenendo Milvus sempre aggiornato.</p>

      </svg>
    </button></h2><p>Poiché Milvus non memorizza i dati grezzi e mantiene solo una mappatura tra i metadati e i dati grezzi, le raccolte esterne sono di sola lettura. Ciò significa che non è possibile inserire, aggiornare, cancellare, importare, svuotare e compattare i dati sul lato Milvus.</p>
<p>Rispetto alle raccolte gestite, le raccolte esterne hanno i seguenti limiti:</p>
<ul>
<li><p>Non è possibile impostare la chiave primaria e gli attributi AutoID.</p></li>
<li><p>Non è possibile abilitare il campo dinamico.</p></li>
<li><p>Non è possibile impostare la chiave di partizione e la chiave di clustering, perché le partizioni non sono disponibili.</p></li>
<li><p>Non si possono definire funzioni nello schema.</p></li>
<li><p>Non è possibile modificare lo schema di una collezione esterna dopo averla creata.</p></li>
<li><p>Non è possibile utilizzare la corrispondenza di testo con BM25.</p></li>
<li><p>È necessario attivare un'operazione di aggiornamento prima di creare gli indici.</p></li>
</ul>

<p>In questo caso, il proprietario del bucket dovrebbe creare un ruolo IAM che</p>
<ul>
<li><p>Allega <code translate="no">AmazonS3FullAccess</code> o un criterio più dettagliato per l'accesso al bucket.</p></li>

</ul>

<p>L'operazione di aggiornamento è asincrona, quindi è necessario impostare un'iterazione per monitorare i suoi progressi.</p>
<div class="alert note">
<ul>
<li><p>L'operazione di aggiornamento analizza i metadati dei file di dati e genera i file manifest di conseguenza. Di solito richiede 150-250 ms.</p></li>
<li><p>I file manifest registrano la mappatura tra i metadati di Milvus e le righe dei file esterni.</p></li>
<li><p>In caso di aggiornamento dei dati di origine, è necessario richiamare manualmente l'aggiornamento per mantenere Milvus aggiornato.</p></li>
<li><p>Non è possibile indicizzare una collezione esterna solo al termine dell'aggiornamento. Tuttavia, la modalità di creazione degli indici è la stessa di una collezione gestita.</p></li>
<li><p>Un aggiornamento che richiede la rimozione di tutti i metadati attivi senza alcun inserimento comporta un rifiuto.</p></li>
</ul>
