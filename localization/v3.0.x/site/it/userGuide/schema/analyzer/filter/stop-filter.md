     <td><p><code translate="no">stop_words</code></p></td>
     <td><p>Un elenco di parole da rimuovere dalla tokenizzazione. Per impostazione predefinita, il filtro utilizza il dizionario integrato <code translate="no">_english_</code>. È possibile sovrascriverlo o estenderlo in tre modi:</p><ul><li><p><strong>Dizionari incorporati</strong> - fornire uno di questi alias di lingua per utilizzare un dizionario predefinito:</p><p><code translate="no">"_english_"</code>, <code translate="no">"_danish_"</code>, <code translate="no">"_dutch_"</code>, <code translate="no">"_finnish_"</code>, <code translate="no">"_french_"</code>, <code translate="no">"_german_"</code>, <code translate="no">"_hungarian_"</code>, <code translate="no">"_italian_"</code>, <code translate="no">"_norwegian_"</code>, <code translate="no">"_portuguese_"</code>, <code translate="no">"_russian_"</code>, <code translate="no">"_spanish_"</code>, <code translate="no">"_swedish_"</code></p></li><li><p><strong>Elenco personalizzato</strong> - passare un array di termini personalizzati, ad esempio <code translate="no">["foo", "bar", "baz"]</code>.</p></li><li><p><strong>Elenco misto</strong> - combina alias e termini personalizzati, ad esempio <code translate="no">["of", "to", "_english_"]</code>.</p><p>Per i dettagli sul contenuto esatto di ciascun dizionario predefinito, fare riferimento a <a href="https://github.com/milvus-io/milvus/blob/master/internal/core/thirdparty/tantivy/tantivy-binding/src/analyzer/filter/stop_words.rs">stop_words</a>.</p></li></ul></td>
   </tr>
</table>
<p>Il filtro <code translate="no">stop</code> opera sui termini generati dal tokenizzatore, quindi deve essere usato in combinazione con un tokenizzatore. Per un elenco dei tokenizer disponibili in Milvus, si rimanda a <a href="/docs/it/standard-tokenizer.md">Standard Tokenizer</a> e alle sue pagine collaterali.</p>
<p>Dopo aver definito <code translate="no">analyzer_params</code>, è possibile applicarlo a un campo <code translate="no">VARCHAR</code> quando si definisce uno schema di raccolta. Ciò consente a Milvus di elaborare il testo in quel campo usando l'analizzatore specificato per una tokenizzazione e un filtraggio efficienti. Per i dettagli, vedere <a href="/docs/it/analyzer-overview.md#Example-use">Esempio di utilizzo</a>.</p>

    </button></h3><p>Per elenchi di stop-words personalizzati di grandi dimensioni (elenchi specifici di una lingua, vocabolari di dominio o elenchi che si desidera condividere in molte raccolte), è possibile memorizzare le parole in un file e registrare il file come risorsa di file remota, quindi fare riferimento ad esso dal filtro mediante il parametro <code translate="no">stop_words_file</code>. È possibile utilizzare <code translate="no">stop_words_file</code> da solo o insieme a <code translate="no">stop_words</code> inline; quando entrambi sono impostati, il filtro fonde le due fonti in un unico elenco di stop-words.</p>
