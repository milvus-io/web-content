        ></path>
      </svg>
    </button></h3><p>Prima di creare un'istantanea, si consiglia di interrompere la scrittura dei dati nella raccolta di destinazione e di chiamare <code translate="no">flush()</code> per evitare una possibile perdita di dati.</p>
<div class="alert note">
<p>La chiamata a <code translate="no">flush()</code> non è obbligatoria, ma è altamente consigliata per evitare la perdita di dati. Se si salta questa operazione, l'istantanea contiene solo i dati che sono già stati scaricati.</p>
</div>
<p>Quando si assegna un nome a un'istantanea, utilizzare nomi chiari e descrittivi, come <code translate="no">&quot;daily_backup_20240101&quot;</code> o <code translate="no">&quot;v2.1_production_release&quot;</code> ed evitare termini generici, come <code translate="no">&quot;backup1&quot;</code> e <code translate="no">&quot;test&quot;</code>. Usate saggiamente i nomi delle istantanee per distinguere le istantanee tra le varie versioni, ambienti e fasi.</p>
<p>Gli esempi di codice che seguono presuppongono che si abbia già un insieme chiamato <code translate="no">my_collection</code>.</p>

      </svg>
    </button></h3><p>È possibile ripristinare un'istantanea in una nuova raccolta. Questa operazione è asincrona e restituisce un ID lavoro per tracciare l'avanzamento del ripristino.</p>
<p>Il ripristino utilizza un meccanismo di <strong>copia dei segmenti</strong> al posto dell'importazione dei dati, che è più efficiente in quanto</p>
<ul>
<li><p>copia direttamente i file di segmento (binlog, deltalog, file di indice) dalla memoria snapshot</p></li>
<li><p>conserva gli ID dei campi e gli ID degli indici per garantire la compatibilità con i file di dati esistenti</p></li>
<li><p>evita la riscrittura dei dati e la ricostruzione degli indici, con conseguenti tempi di ripristino significativamente più rapidi, e</p></li>
<li><p>garantisce un aumento delle prestazioni da 10 a 100 volte rispetto ai metodi di backup e ripristino tradizionali.</p></li>
</ul>
<p>Per ripristinare un'istantanea, procedere come segue:</p>
