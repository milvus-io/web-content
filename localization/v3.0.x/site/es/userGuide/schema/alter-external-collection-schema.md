    </button></h1><p>Las fuentes de datos externas suelen evolucionar después de crear una colección externa. Por ejemplo, una tabla de «lakehouse» que ya almacena representaciones vectoriales podría incluir posteriormente un nuevo campo escalar, como una puntuación, una categoría o una marca de tiempo, que desees incluir en los resultados de las consultas o utilizar en los filtros.</p>
<p>En lugar de volver a crear la colección externa o copiar los datos de origen en Milvus, añade un campo de Milvus que se corresponda con el campo existente en la fuente de datos externa. Tras añadir el campo, actualiza la colección externa para que el nuevo campo pueda utilizarse en consultas y búsquedas.</p>

      </svg>
    </button></h2><ul>
<li><p>Actualmente, las colecciones externas admiten la adición de campos tras su creación. No se admiten otros cambios en el esquema, como eliminar campos, renombrarlos, cambiar los tipos de datos de los campos, modificar las dimensiones de los vectores o reasignar <code translate="no">external_field</code>.</p></li>
<li><p>Solo se puede añadir un campo que ya exista en la fuente de datos externa. Esta operación asigna un campo externo existente a un campo de Milvus. No crea un nuevo campo en la fuente de datos externa ni rellena los datos de origen.</p></li>
<li><p>No se admite la adición de campos de tipo « <code translate="no">SPARSE_FLOAT_VECTOR</code> » a una colección externa existente.</p></li>
<li><p>No se admite la adición de campos «StructArray» a una colección externa existente. Si su colección externa necesita un campo «StructArray», defínalo en el esquema de la colección al crearla.</p></li>
</ul>

      </svg>
    </button></h2><p>Antes de añadir un campo a una colección externa, comprueba que el campo ya exista en la fuente de datos externa. A continuación, llama a ` <code translate="no">add_collection_field()</code> ` para exponer ese campo en Milvus, estableciendo ` <code translate="no">external_field</code> ` en el nombre del campo de la fuente de datos externa. Establece ` <code translate="no">data_type</code> ` en el tipo de datos de Milvus que coincida con el campo de la fuente de datos externa. Por ejemplo, si el campo mapeado almacena valores de doble precisión, utiliza ` <code translate="no">DataType.DOUBLE</code>`.</p>
<p>A diferencia de las colecciones gestionadas, los valores del campo añadido se leen de la fuente de datos externa tras actualizar la colección externa.</p>

    </button></h3><p>Utilice <code translate="no">add_collection_field()</code> para añadir un campo escalar cuando desee que dicho campo aparezca en los resultados de la consulta o utilizarlo en filtros. El siguiente ejemplo añade un campo <code translate="no">score</code> que se asigna al campo <code translate="no">score</code> de la fuente de datos externa.</p>
<pre><code translate="no" class="language-python"><span class="hljs-keyword">from</span> pymilvus <span class="hljs-keyword">import</span> DataType, MilvusClient

client = MilvusClient(
    uri=<span class="hljs-string">&quot;http://localhost:19530&quot;</span>,
    token=<span class="hljs-string">&quot;root:Milvus&quot;</span>,
)

client.add_collection_field(
    collection_name=<span class="hljs-string">&quot;product_embeddings&quot;</span>,
    field_name=<span class="hljs-string">&quot;score&quot;</span>,
    data_type=DataType.DOUBLE,
    nullable=<span class="hljs-literal">True</span>,
<span class="highlighted-wrapper-line">    external_field=<span class="hljs-string">&quot;score&quot;</span>,</span>
)
<button class="copy-code-btn"></button></code></pre>
<p>En este ejemplo, « <code translate="no">score</code> » es el nombre del campo de Milvus y « <code translate="no">external_field=&quot;score&quot;</code> » lo asigna al campo « <code translate="no">score</code> » de la fuente de datos externa. Establece « <code translate="no">nullable=True</code> », ya que el campo se añade después de que la colección ya se haya creado.</p>
