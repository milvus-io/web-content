    </button></h1><p>Le filtre <code translate="no">synonym</code> réécrit les mots en fonction d'un dictionnaire de synonymes, de sorte que les termes apparentés correspondent lors de la recherche. Il prend en charge deux modes de fonctionnement et deux façons de fournir le dictionnaire :</p>
<ul>
<li><p><strong>Modes de fonctionnement</strong> - le mode <code translate="no">expand</code> préserve l'élément original et émet des synonymes supplémentaires en même temps que lui ; le mode de normalisation (<code translate="no">expand: false</code>) réécrit les éléments sous une forme canonique.</p></li>
<li><p><strong>Sources du dictionnaire</strong> - les petits dictionnaires peuvent être intégrés dans la configuration du filtre via le tableau <code translate="no">synonyms</code>; les grands dictionnaires doivent être stockés dans un <a href="/docs/fr/manage-file-resources.md">fichier</a> et référencés via <code translate="no">synonyms_file</code>.</p></li>
</ul>

<p>Les mots-clés de gauche (<code translate="no">fast</code>, <code translate="no">quick</code>) se réécrivent en mots-clés de droite (<code translate="no">speedy</code>). Plusieurs cibles sont autorisées :</p>
<pre><code translate="no" class="language-plaintext">small, little =&gt; tiny, compact
<button class="copy-code-btn"></button></code></pre>
<p>Avec <code translate="no">expand: true</code>, les jetons d'origine sont conservés à côté des cibles :</p>
<ul>
<li><p>Entrée <code translate="no">fast</code> avec <code translate="no">expand: true</code> → <code translate="no">fast</code>, <code translate="no">speedy</code></p></li>
<li><p>Entrée <code translate="no">fast</code> avec <code translate="no">expand: false</code> → <code translate="no">speedy</code></p></li>
</ul>

<p>Tous les éléments énumérés sont considérés comme équivalents :</p>
<ul>
<li><p>Avec <code translate="no">expand: true</code>, toute occurrence d'un token du groupe émet tous les tokens du groupe. Entrée <code translate="no">happy</code> → <code translate="no">happy</code>, <code translate="no">joyful</code>, <code translate="no">cheerful</code>.</p></li>
<li><p>Avec <code translate="no">expand: false</code>, chaque occurrence est réécrite en premier token du groupe. Entrée <code translate="no">joyful</code> → <code translate="no">happy</code>; l'entrée <code translate="no">happy</code> est déjà le premier token et reste inchangée.</p></li>
</ul>

<p>Le filtre <code translate="no">synonym</code> accepte les paramètres suivants.</p>
<table>
   <tr>
     <th><p><strong>Paramètre</strong></p></th>
     <th><p><strong>Description</strong></p></th>
     <th><p><strong>Défaut</strong></p></th>
   </tr>
   <tr>
     <td><p><code translate="no">synonyms</code></p></td>
     <td><p>Un tableau de chaînes de règles en ligne. Chaque chaîne utilise le format de dictionnaire décrit ci-dessus. Convient aux petits dictionnaires (jusqu'à quelques dizaines de règles).</p></td>
     <td><p>-</p></td>
   </tr>
   <tr>
     <td><p><code translate="no">synonyms_file</code></p></td>
     <td><p>Une référence à un <a href="/docs/fr/manage-file-resources.md">fichier de ressources</a> qui stocke les règles de synonymes, une par ligne. À utiliser pour les dictionnaires plus importants. Voir <a href="/docs/fr/synonym-filter.md#External-dictionary-file">Fichier de dictionnaire externe</a> ci-dessous.</p></td>
     <td><p>-</p></td>
   </tr>
   <tr>
     <td><p><code translate="no">expand</code></p></td>
     <td><p>Un indicateur booléen qui contrôle la manière dont les règles s'appliquent. true préserve l'élément original et émet des synonymes à côté ; false réécrit les éléments à leur forme canonique (le côté droit d'une correspondance, ou le premier élément d'un groupe d'équivalence).</p></td>
     <td><p>false</p></td>
   </tr>
</table>
<p>Vous pouvez spécifier <code translate="no">synonyms</code>, <code translate="no">synonyms_file</code>, ou les deux. Si les deux sont présents, le filtre fusionne les deux sources. Le filtre opère sur les jetons produits par le tokenizer ; il doit donc être combiné avec un tokenizer tel que le tokenizer <a href="/docs/fr/standard-tokenizer.md">standard</a>.</p>
