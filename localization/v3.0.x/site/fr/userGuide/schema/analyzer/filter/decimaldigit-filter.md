Cela permet d’assurer la cohérence des tokens numériques entre les langues et les systèmes d’écriture.</p>

<p>Pour le texte arabe, l’analyseur intégré <a href="/docs/fr/arabic-analyzer.md"><code translate="no">arabic</code></a> analyseur intégré inclut déjà le filtre « <code translate="no">decimaldigit</code> ». </div>
<p>Pour utiliser le filtre « <code translate="no">decimaldigit</code> » dans un analyseur personnalisé, ajoutez-le à la section « <code translate="no">filter</code> » dans <code translate="no">analyzer_params</code>:</p>
<pre><code translate="no" class="language-python">analyzer_params = {
    <span class="hljs-string">&quot;tokenizer&quot;</span>: <span class="hljs-string">&quot;standard&quot;</span>,
    <span class="hljs-string">&quot;filter&quot;</span>: [<span class="hljs-string">&quot;decimaldigit&quot;</span>],
}
<button class="copy-code-btn"></button></code></pre>
<p>Le filtre « <code translate="no">decimaldigit</code> » ne comporte aucun paramètre configurable.</p>
<p>Le filtre convertit les chiffres décimaux Unicode, y compris les chiffres arabo-indiens, thaï, devanagari, bengali et à pleine largeur, en chiffres ASCII. Il agit sur les tokens générés par le tokenizer. La configuration ci-dessus est volontairement un exemple d’analyseur personnalisé et n’inclut pas le pipeline complet de traitement de l’arabe.</p>
