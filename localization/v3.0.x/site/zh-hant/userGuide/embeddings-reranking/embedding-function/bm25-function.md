    </button></h1><p><strong>BM25 函式</strong>可將原始文字轉換為<strong>稀疏向量</strong>，並根據詞彙相關性對文件進行評分，從而實現<a href="/docs/zh-hant/full-text-search.md">全文本搜尋</a>。它應用了基於詞彙的匹配和頻率感知權重，以支援與查詢詞彙密切匹配的文本文檔的高效檢索。</p>
<p>作為一個本機文字函式，BM25 函式在 Milvus 內執行，不需要模型推論或外部整合。它為基於文字的搜尋情況提供了確定且透明的檢索機制。</p>

    </button></h2><p><a href="https://en.wikipedia.org/wiki/Okapi_BM25">BM25</a>演算法是一種基於詞彙的相關性評分演算法，廣泛應用於全文檢索。在 Milvus 中，BM25 實作為稀疏檢索管道，可將文字轉換為詞彙權重表示，並使用分散式稀疏索引檢索前<em>K 個</em>文件。</p>
<p>整體工作流程包含兩個對稱的路徑：<strong>文件擷取</strong>和<strong>查詢文字處理</strong>，它們共用相同的文字分析邏輯。</p>

      </svg>
    </button></h3><p>插入文件時，其原始文字會先由<strong><a href="/docs/zh-hant/analyzer-overview.md">分析器</a></strong>處理，<strong><a href="/docs/zh-hant/analyzer-overview.md">分析器</a></strong>會將文字標記化為個別術語。</p>
<p>例如，文件：</p>
<pre><code translate="no" class="language-plaintext">&quot;We are loving Milvus!&quot;
<button class="copy-code-btn"></button></code></pre>
<p>可分析為下列詞彙：</p>
<pre><code translate="no" class="language-plaintext">[&quot;we&quot;, &quot;love&quot;, &quot;milvus&quot;]
<button class="copy-code-btn"></button></code></pre>
<p>然後，每個文件會以詞彙頻率 (TF) 表示，記錄每個詞彙在文件中出現的次數。舉例來說</p>
<pre><code translate="no" class="language-plaintext">{
  &quot;we&quot;: 1,
  &quot;love&quot;: 1,
  &quot;milvus&quot;: 1
}
<button class="copy-code-btn"></button></code></pre>
<p>同時，Milvus 會更新語料庫層級的統計資料，包括</p>
<ul>
<li><p>每個詞彙的文檔頻率 (DF)</p></li>
<li><p>平均文件長度</p></li>
<li><p>將每個詞彙映射到包含該詞彙的文檔的發佈清單</p></li>
</ul>
<p>將文件的 TF 表示法插入<strong>稀疏嵌入（sparse embeddings</strong>），其中的詞彙發佈會在節點間進行分割，以便進行可擴展性檢索。</p>

    </button></h3><p>當發出基於文字的查詢時，會由在<a href="/docs/zh-hant/bm25-function.md#Document-ingestion-From-text-to-sparse-representation">文件擷取</a>過程中使用的<strong>相同分析器</strong>來處理，以確保術語分割的一致性。</p>
<p>例如，查詢</p>
<pre><code translate="no" class="language-plaintext">&quot;who loves Milvus?&quot;
<button class="copy-code-btn"></button></code></pre>
<p>可以被分析成</p>
<pre><code translate="no" class="language-plaintext">[&quot;who&quot;, &quot;love&quot;, &quot;milvus&quot;]
<button class="copy-code-btn"></button></code></pre>
<p>對於每個查詢詞彙，Milvus 會從語料庫統計中查詢其<a href="https://en.wikipedia.org/wiki/Tf%E2%80%93idf">反向文檔頻率</a>(IDF)。IDF 反映了一個詞彙在整個資料集中的資訊量：較罕見的詞彙會得到較高的權重，而常見的詞彙會得到較低的權重。</p>
<p>在概念上，這會產生一組 IDF 加權的查詢詞，例如：</p>
<pre><code translate="no" class="language-plaintext">{
  &quot;who&quot;: 0.1,
  &quot;love&quot;: 0.5,
  &quot;milvus&quot;: 1.2
}
<button class="copy-code-btn"></button></code></pre>
<h3 id="BM25-scoring-and-top-K-retrieval" class="common-anchor-header">BM25 評分與前 K 檢索<button data-href="#BM25-scoring-and-top-K-retrieval" class="anchor-icon" translate="no">

          d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"
        ></path>
      </svg>
    </button></h3><p>BM25 根據匹配的查詢字詞計算相關性分數，從而對文件進行排序。計分在<strong>詞彙層級</strong>執行，並在<strong>文件層級</strong>彙總。</p>
<p><strong>詞彙層級計分</strong></p>
<p>對於出現在文件中的每個查詢詞彙，BM25 會計算詞彙層級的得分：</p>
<pre><code translate="no" class="language-plaintext">term_score =
  IDF(term) ×
  TF_boost(term, document, k1) ×
  length_normalization(document, b)
<button class="copy-code-btn"></button></code></pre>
<p>其中：</p>
<ul>
<li><p><strong>IDF(term)</strong>反映該詞彙在集合中的罕見程度</p></li>
<li><p><strong>TF_boost(...,k1)</strong>隨著詞彙頻率的增加而增加，但隨著頻率的增加而飽和</p></li>
<li><p><strong>length_normalization(..., b)</strong>根據文件長度調整得分</p></li>
</ul>
<p><strong>文件層級計分和 Top-K 檢索</strong></p>
<p>最終的文件分數是所有匹配查詢字詞的字詞層級分數的總和：</p>
<pre><code translate="no" class="language-plaintext">document_score =
  sum of term_score over all matched query terms
<button class="copy-code-btn"></button></code></pre>
<p>文檔會依最終得分排序，並傳送得分最高的 Top-K 文檔。</p>

        ></path>
      </svg>
    </button></h2><p>使用 BM25 功能之前，請規劃您的資料庫模式，以確保它支援詞彙全文檢索：</p>
<ul>
<li><p><strong>原始內容的文字欄位</strong></p>
<p>您的資料集中必須包含一個<code translate="no">VARCHAR</code> 欄位來儲存原始文字。這個欄位是將被處理以進行全文檢索的文字來源。</p></li>
<li><p><strong>文字欄位的分析器</strong></p>
<p>文字欄位必須啟用分析器。分析器定義了在 BM25 函式計算詞彙相關性之前，如何將文字標記化和規範化。</p>
<p>預設情況下，Milvus 提供內建分析器，根據空白和標點符號化文字。如果您的應用程式需要自訂標記化或規範化行為，您可以定義一個自訂分析器。詳情請參閱<a href="/docs/zh-hant/choose-the-right-analyzer-for-your-use-case.md">為您的使用個案選擇正確的分析器</a>。</p></li>
<li><p><strong>BM25 輸出的稀疏向量</strong></p>
<p>您的集合必須包含<code translate="no">SPARSE_FLOAT_VECTOR</code> 欄位，以儲存 BM25 函式所產生的稀疏表示。這個欄位用於全文檢索時的索引和檢索。</p></li>
</ul>
<p>想好這些模式層級的注意事項後，請繼續建立資料集並使用 BM25 函式。</p>
