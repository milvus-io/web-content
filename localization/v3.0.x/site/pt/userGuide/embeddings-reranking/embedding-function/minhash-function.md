    </button></h1><p>A <strong>função MinHash</strong> converte texto em bruto em <strong>vectores binários</strong> que aproximam <a href="https://en.wikipedia.org/wiki/Jaccard_index">a similaridade Jaccard</a> entre documentos. Aplica shingling de texto e múltiplas funções de hash para produzir vectores de assinatura de comprimento fixo, permitindo uma rápida deteção de quase-duplicados e desduplicação de documentos em escala.</p>
<p>Como uma função integrada, o MinHash é executado no Milvus e não requer inferência de modelo externo ou pré-processamento. Insere-se texto em bruto e o Milvus gera automaticamente os vectores de assinatura MinHash.</p>

<p><a href="https://en.wikipedia.org/wiki/MinHash">MinHash</a> é uma técnica de hashing sensível à localidade que estima <a href="https://en.wikipedia.org/wiki/Jaccard_index">a similaridade Jaccard</a> entre conjuntos. No Milvus, a função MinHash segue o seguinte fluxo de trabalho: o utilizador fornece texto em bruto como entrada e o Milvus produz um vetor binário como saída - tratando internamente de todos os passos intermédios.</p>
<p>O fluxo de trabalho global consiste num <strong>pipeline de processamento de texto partilhado</strong>, utilizado tanto para a ingestão de documentos como para o processamento de consultas, seguido de operações específicas de cada fase para armazenamento e recuperação.</p>

        ></path>
      </svg>
    </button></h3><p>Tanto a ingestão de documentos como o processamento de consultas passam o texto em bruto pela mesma transformação em quatro fases:</p>
<ol>
<li><p><strong>Análise de texto</strong>: O texto é processado por um <a href="/docs/pt/analyzer-overview.md">analisador</a> (quando <code translate="no">token_level</code> é <code translate="no">&quot;word&quot;</code>) ou utilizado diretamente (quando <code translate="no">token_level</code> é <code translate="no">&quot;char&quot;</code>). A tokenização ao nível da palavra aplica o analisador configurado no campo de entrada para segmentar o texto em termos - por exemplo, <code translate="no">&quot;milvus is vector db&quot;</code> torna-se <code translate="no">[&quot;milvus&quot;, &quot;is&quot;, &quot;vector&quot;, &quot;db&quot;]</code>.</p></li>
<li><p><strong>Shingling</strong>: Os tokens são divididos em n-gramas sobrepostos (shingles) de tamanho <code translate="no">shingle_size</code>. Por exemplo, com 3-gramas ao nível da palavra, os tokens <code translate="no">[&quot;information&quot;, &quot;retrieval&quot;, &quot;is&quot;, &quot;a&quot;, &quot;field&quot;]</code> tornam-se shingles como <code translate="no">[&quot;information retrieval is&quot;, &quot;retrieval is a&quot;, &quot;is a field&quot;]</code>.</p></li>
<li><p><strong>Geração de assinaturas MinHash</strong>: Múltiplas funções de hash (H1, H2, ..., Hn, onde n = <code translate="no">num_hashes</code>) são aplicadas ao conjunto de shingles. Para cada função de hash, é selecionado o valor mínimo de hash em todos os shingles. A coleção destes valores mínimos forma a assinatura MinHash - uma representação de comprimento fixo que aproxima a semelhança Jaccard do documento original.</p></li>
<li><p><strong>Codificação de vetor binário</strong>: Cada valor de assinatura é um hash de 32 bits, e a assinatura completa é compactada num <code translate="no">BINARY_VECTOR</code> de dimensão <code translate="no">32 * num_hashes</code>.</p></li>
</ol>

    </button></h3><p>Durante a pesquisa, o texto da consulta passa pelo mesmo pipeline partilhado para produzir um vetor binário. Este vetor é utilizado para efetuar uma pesquisa LSH no índice <code translate="no">MINHASH_LSH</code>, que identifica rapidamente os pares de candidatos que são provavelmente semelhantes. Os candidatos são então classificados por similaridade Jaccard estimada e os resultados top-K são retornados.</p>
<p>Como ambos os caminhos partilham a mesma lógica de transformação, dois documentos com conteúdo altamente sobreposto produzem assinaturas MinHash semelhantes. Isso torna a função eficaz para encontrar quase duplicatas, mesmo quando os documentos diferem em ordem de palavras, formatação ou frases menores.</p>

    </button></h2><p>Antes de usar a função MinHash, planeje seu esquema de coleção para incluir o seguinte:</p>
<ul>
<li><p><strong>Um campo de texto para conteúdo bruto</strong></p>
<p>Sua coleção deve incluir um campo <code translate="no">VARCHAR</code> para armazenar texto bruto. Este campo serve como entrada para a função MinHash.</p></li>
<li><p><strong>Um analisador para o campo de texto</strong> (ao usar a tokenização no nível da palavra)</p>
<p>Se <code translate="no">token_level</code> estiver definido como <code translate="no">&quot;word&quot;</code> (predefinição), o campo de texto deve ter um analisador ativado. O analisador define como o texto é tokenizado antes do shingling. Por defeito, o Milvus utiliza o analisador <code translate="no">standard</code>. Para configurar um analisador diferente, consulte <a href="/docs/pt/choose-the-right-analyzer-for-your-use-case.md">Escolha o analisador certo para o seu caso de uso</a>.</p></li>
<li><p><strong>Um campo de vetor binário para saída MinHash</strong></p>
<p>Sua coleção deve incluir um campo <code translate="no">BINARY_VECTOR</code> para armazenar os vetores binários gerados pela função MinHash. A dimensão deve ser igual a <code translate="no">32 * num_hashes</code>.</p></li>
</ul>
