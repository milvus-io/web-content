    </button></h2><h3 id="Increase-resources-per-MinIO-pod" class="common-anchor-header">Увеличение ресурсов на одну капсулу MinIO</h3><p>MinIO, система хранения объектов, используемая Milvus, может увеличить ресурсы процессора и памяти для каждого стручка.</p>
<pre><code translate="no" class="language-yaml"><span class="hljs-comment"># new-values.yaml</span>
minio:
  resources:
     limits:
       cpu: <span class="hljs-number">2</span>
       memory: 8Gi
<button class="copy-code-btn"></button></code></pre>
<p>После сохранения файла примените изменения с помощью следующей команды:</p>
<pre><code translate="no" class="language-shell">helm upgrade &lt;milvus-release&gt; --reuse-values -f <span class="hljs-keyword">new</span>-values.<span class="hljs-property">yaml</span> milvus/milvus
<button class="copy-code-btn"></button></code></pre>
<p>Вы также можете увеличить дисковую емкость кластера MioIO, вручную изменив значение <code translate="no">spec.resources.requests.storage</code> для каждого MioIO Persistent Volume Claim (PVC). Обратите внимание, что класс хранения по умолчанию должен позволять расширение тома.</p>
<h3 id="Add-an-extra-MinIO-server-pool-Recommended" class="common-anchor-header">Добавьте дополнительный пул серверов MinIO (рекомендуется)</h3><p>Рекомендуется добавить дополнительный пул серверов MioIO для экземпляра Milvus.</p>
<pre><code translate="no" class="language-yaml"><span class="hljs-comment"># new-values.yam;</span>
minio:
  zones: <span class="hljs-number">2</span>
<button class="copy-code-btn"></button></code></pre>
<p>После сохранения файла примените изменения с помощью следующей команды:</p>
<pre><code translate="no" class="language-shell">helm upgrade &lt;milvus-release&gt; --reuse-values -f <span class="hljs-keyword">new</span>-values.<span class="hljs-property">yaml</span> milvus/milvus
<button class="copy-code-btn"></button></code></pre>
<p>Это добавит дополнительный пул серверов в ваш кластер MinIO, что позволит Milvus записывать данные в пул серверов MinIO на основе свободного дискового пространства каждого пула серверов. Например, если группа из трех пулов имеет в общей сложности 10 ТБ свободного пространства, распределенного между пулами следующим образом:</p>
<table>
<thead>
<tr><th></th><th>Свободное пространство</th><th>Возможность записи</th></tr>
</thead>
<tbody>
<tr><td>Пул A</td><td>3 Тб</td><td>30% (3/10)</td></tr>
<tr><td>Пул B</td><td>2 ТиБ</td><td>20% (2/10)</td></tr>
<tr><td>Пул C</td><td>5 ТиБ</td><td>50% (5/10)</td></tr>
</tbody>
</table>
<div class="alert note">
<p>MinIO не производит автоматическую ребалансировку объектов в новых пулах серверов. При необходимости вы можете вручную инициировать процедуру ребалансировки с помощью <code translate="no">mc admin rebalance</code>.</p>
</div>
