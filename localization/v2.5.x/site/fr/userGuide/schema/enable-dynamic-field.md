    </button></h2><p>Dans Milvus, vous pouvez créer un schéma de collection en définissant les noms et les types de données de chaque champ de la collection. Lorsque vous ajoutez un champ au schéma, assurez-vous que ce champ est inclus dans l'entité que vous avez l'intention d'insérer. Si vous souhaitez que certains champs soient facultatifs, vous pouvez activer le champ dynamique.</p>
<p>Le champ dynamique est un champ réservé nommé <strong>$meta</strong>, qui est de type JavaScript Object Notation (JSON). Tous les champs des entités qui ne sont pas définis dans le schéma seront stockés dans ce champ JSON réservé sous forme de paires clé-valeur.</p>
<p>Pour une collection dont le champ dynamique est activé, vous pouvez utiliser les clés du champ dynamique pour le filtrage scalaire, comme vous le feriez avec des champs explicitement définis dans le schéma.</p>

<h3 id="Index-a-scalar-field-in-the-dynamic-field" class="common-anchor-header">Indexer un champ scalaire dans le champ dynamique</h3><p>Lorsque vous activez un champ dynamique, tous les champs scalaires non définis sont stockés sous forme de paires clé-valeur au format JSON. Milvus prend en charge la création d'un index sur un tel champ scalaire non défini, en construisant un index de chemin JSON. Voici comment cela fonctionne :</p>
<ol>
<li><p><strong>Choisissez la clé du champ dynamique</strong> que vous souhaitez indexer. Par exemple, <code translate="no">&quot;color&quot;</code> dans l'exemple ci-dessus.</p></li>
<li><p><strong>Décidez d'un type de distribution</strong> pour les valeurs trouvées à cette clé. Milvus analyse le champ dynamique, extrait les valeurs sous la clé spécifiée et les coule dans le type que vous avez configuré.</p>
<ul>
<li><p>Les valeurs <code translate="no">json_cast_type</code> prises en charge sont <code translate="no">bool</code> (ou <code translate="no">BOOL</code>), <code translate="no">double</code> (ou <code translate="no">DOUBLE</code>) et <code translate="no">varchar</code> (ou <code translate="no">VARCHAR</code>).</p></li>
<li><p>Si l'analyse ou la conversion échoue (par exemple, si l'on essaie d'analyser une chaîne en tant que double), ces lignes seront ignorées dans l'index.</p></li>
</ul></li>
<li><p><strong>Spécifiez le chemin JSON</strong> vers cette clé sous la forme <code translate="no">json_path</code>. Étant donné que le champ dynamique est stocké sous forme de JSON, vous pouvez spécifier quelque chose comme <code translate="no">&quot;color&quot;</code>, ou si vous avez des structures imbriquées, vous pouvez spécifier des chemins plus profonds (par exemple <code translate="no">my_json[&quot;field&quot;][&quot;subfield&quot;]</code>).</p></li>
<li><p><strong>Créez un index INVERTED</strong>. Actuellement, seul le type <code translate="no">INVERTED</code> est pris en charge pour l'indexation des chemins JSON.</p></li>
</ol>
<p>Pour plus de détails sur les paramètres et les considérations, voir <a href="/docs/fr/use-json-fields.md#Index-a-JSON-field">Indexer un champ JSON</a>.</p>
