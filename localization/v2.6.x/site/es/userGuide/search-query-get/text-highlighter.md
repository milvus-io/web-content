    </button></h1><p>El Resaltador de Milvus anota los términos coincidentes en los campos de texto envolviéndolos con etiquetas personalizables. El resaltado ayuda a explicar por qué un documento coincide, mejora la legibilidad de los resultados y permite una representación enriquecida en las aplicaciones de búsqueda y RAG.</p>
<p>El resaltado se ejecuta como un paso de posprocesamiento en el conjunto final de resultados de búsqueda. No afecta a la recuperación de candidatos, la lógica de filtrado, la clasificación ni la puntuación.</p>
<p>El Resaltador ofrece tres dimensiones independientes de control:</p>
<ul>
<li><p><strong>Qué términos se resaltan</strong></p>
<p>Puede elegir de dónde proceden los términos resaltados. Por ejemplo, resalte los términos de búsqueda utilizados en <strong>la búsqueda de texto completo de BM25</strong>, o los términos de consulta especificados en <strong>expresiones de filtrado basadas en texto</strong> (como las condiciones de <code translate="no">TEXT_MATCH</code> ).</p></li>
<li><p><strong>Cómo se muestran los términos resaltados</strong></p>
<p>Puede controlar cómo aparecen los términos coincidentes en la salida de resaltado configurando las etiquetas insertadas antes y después de cada coincidencia. Por ejemplo, utilice marcadores sencillos como <code translate="no">{}</code> o etiquetas HTML como <code translate="no">&lt;em&gt;&lt;/em&gt;</code> para una representación enriquecida.</p></li>
<li><p><strong>Cómo se devuelve el texto resaltado</strong></p>
<p>Puede controlar cómo se devuelven los resultados resaltados como fragmentos, incluyendo dónde empiezan los fragmentos, qué longitud tienen y cuántos fragmentos se devuelven.</p></li>
</ul>
<p>En las secciones siguientes se describen estas opciones.</p>

    </button></h2><p>Cuando realice una búsqueda de texto completo en BM25, puede resaltar los <strong>términos de búsqueda</strong> en el resultado devuelto para ayudar a explicar por qué un documento coincide con la consulta. Para obtener más información sobre la búsqueda de texto completo en BM25, consulte <a href="/docs/es/full-text-search.md">Búsqueda de texto completo</a>.</p>
<p>En este escenario, los términos resaltados proceden directamente de los términos de búsqueda utilizados en la búsqueda de texto completo BM25. El Resaltador utiliza estos términos para anotar el texto coincidente en el resultado final.</p>
<p>Supongamos que el siguiente contenido está almacenado en un campo de texto:</p>
<pre><code translate="no" class="language-plaintext">Milvus supports full text search. Use BM25 for keyword relevance. Filters can narrow results.
<button class="copy-code-btn"></button></code></pre>
<p><strong>Configuración del Resaltador</strong></p>
<p>Para resaltar los términos de búsqueda en la búsqueda de texto completo BM25, cree un <code translate="no">LexicalHighlighter</code> y active el resaltado de términos de búsqueda para la búsqueda de texto completo BM25:</p>
<pre><code translate="no" class="language-python"><span class="hljs-keyword">from</span> pymilvus <span class="hljs-keyword">import</span> LexicalHighlighter

highlighter = LexicalHighlighter(
    pre_tags=[<span class="hljs-string">&quot;{&quot;</span>],              <span class="hljs-comment"># Tag inserted before each highlighted term</span>
    post_tags=[<span class="hljs-string">&quot;}&quot;</span>],             <span class="hljs-comment"># Tag inserted after each highlighted term</span>
    highlight_search_text=<span class="hljs-literal">True</span>   <span class="hljs-comment"># Enable search term highlighting for BM25 full text search</span>
)
<button class="copy-code-btn"></button></code></pre>
<p>En este ejemplo</p>
<ul>
<li><p><code translate="no">pre_tags</code> y <code translate="no">post_tags</code> controlan cómo aparece el texto resaltado en la salida. En este caso, los términos coincidentes se incluyen en <code translate="no">{}</code> (por ejemplo, <code translate="no">{term}</code>). También puede proporcionar varias etiquetas en forma de lista (por ejemplo, <code translate="no">[&quot;&lt;b&gt;&quot;, &quot;&lt;i&gt;&quot;]</code>). Cuando se resaltan varios términos, las etiquetas se aplican en orden y se giran según la secuencia de coincidencia.</p></li>
<li><p><code translate="no">highlight_search_text=True</code> indica a Milvus que utilice los términos de búsqueda en la búsqueda de texto completo BM25 como fuente de términos resaltados.</p></li>
</ul>
<p>Una vez creado el objeto Resaltador, aplique su configuración a su petición de búsqueda de texto completo BM25:</p>
<pre><code translate="no" class="language-python">results = client.search(
    ...,
    data=[<span class="hljs-string">&quot;BM25&quot;</span>],      <span class="hljs-comment"># Search term used in BM25 full text search</span>
<span class="highlighted-wrapper-line">    highlighter=highlighter <span class="hljs-comment"># Pass highlighter config here</span></span>
)
<button class="copy-code-btn"></button></code></pre>
<p><strong>Salida de resaltado</strong></p>
<p>Cuando el resaltado está activado, Milvus devuelve el texto resaltado en un campo dedicado <code translate="no">highlight</code>. Por defecto, la salida resaltada se devuelve como un fragmento a partir del primer término encontrado.</p>
<p>En este ejemplo, el término buscado es <code translate="no">&quot;BM25&quot;</code>, por lo que aparece resaltado en el resultado devuelto:</p>
<pre><code translate="no" class="language-json"><span class="hljs-punctuation">{</span>
    ...<span class="hljs-punctuation">,</span>
    <span class="hljs-attr">&quot;highlight&quot;</span><span class="hljs-punctuation">:</span> <span class="hljs-punctuation">{</span>
        <span class="hljs-attr">&quot;text&quot;</span><span class="hljs-punctuation">:</span> <span class="hljs-punctuation">[</span>
            <span class="hljs-string">&quot;{BM25} for keyword relevance. Filters can narrow results.&quot;</span>
        <span class="hljs-punctuation">]</span>
    <span class="hljs-punctuation">}</span>
<span class="hljs-punctuation">}</span>
<button class="copy-code-btn"></button></code></pre>
<p>Para controlar la posición, la longitud y el número de fragmentos devueltos, consulte <a href="/docs/es/text-highlighter.md#Fragment-based-highlighting-output">Devolver el texto resaltado como fragmentos</a>.</p>

<p>Actualmente, sólo se admite la condición de filtrado <code translate="no">TEXT_MATCH</code> para resaltar términos de consulta. Para obtener más información, consulte <a href="/docs/es/keyword-match.md">Coincidencia de texto</a>.</p>

<p>En este caso, los términos resaltados proceden de expresiones de filtrado basadas en texto. El filtrado determina qué documentos coinciden, mientras que el Resaltador anota los espacios de texto coincidentes.</p>
<p>Supongamos que el siguiente contenido se almacena en un campo de texto:</p>
<pre><code translate="no" class="language-python">This document explains how text filtering works <span class="hljs-keyword">in</span> Milvus.

<p><strong>Configuración del Resaltador</strong></p>
<p>Para resaltar los términos de consulta utilizados en el filtrado, cree un <code translate="no">LexicalHighlighter</code> y defina un <code translate="no">highlight_query</code> que corresponda a la condición de filtrado:</p>
<pre><code translate="no" class="language-python"><span class="hljs-keyword">from</span> pymilvus <span class="hljs-keyword">import</span> LexicalHighlighter

highlighter = LexicalHighlighter(
    pre_tags=[<span class="hljs-string">&quot;{&quot;</span>],              <span class="hljs-comment"># Tag inserted before each highlighted term</span>
    post_tags=[<span class="hljs-string">&quot;}&quot;</span>],             <span class="hljs-comment"># Tag inserted after each highlighted term</span>
    highlight_query=[{
        <span class="hljs-string">&quot;type&quot;</span>: <span class="hljs-string">&quot;TextMatch&quot;</span>,     <span class="hljs-comment"># Text filtering type</span>
        <span class="hljs-string">&quot;field&quot;</span>: <span class="hljs-string">&quot;text&quot;</span>,         <span class="hljs-comment"># Target text field</span>
        <span class="hljs-string">&quot;text&quot;</span>: <span class="hljs-string">&quot;text filtering&quot;</span> <span class="hljs-comment"># Terms to highlight</span>
    }]
)
<button class="copy-code-btn"></button></code></pre>
<p>En esta configuración</p>
<ul>
<li><p><code translate="no">pre_tags</code> y <code translate="no">post_tags</code> controlan cómo aparece el texto resaltado en la salida. En este caso, los términos coincidentes están envueltos por <code translate="no">{}</code> (por ejemplo, <code translate="no">{term}</code>). También puede proporcionar varias etiquetas en forma de lista (por ejemplo, <code translate="no">[&quot;&lt;b&gt;&quot;, &quot;&lt;i&gt;&quot;]</code>). Cuando se resaltan varios términos, las etiquetas se aplican en orden y se giran según la secuencia de coincidencia.</p></li>
<li><p><code translate="no">highlight_query</code> define qué términos de filtrado deben resaltarse.</p></li>
</ul>
<p>Una vez creado el objeto Resaltador, aplique la misma expresión de filtrado y la configuración del resaltador a su petición de búsqueda:</p>
<pre><code translate="no" class="language-python">results = client.search(
    ...,
    <span class="hljs-built_in">filter</span>=<span class="hljs-string">&#x27;TEXT_MATCH(text, &quot;text filtering&quot;)&#x27;</span>,
<span class="highlighted-wrapper-line">    highlighter=highlighter <span class="hljs-comment"># Pass highlighter config here</span></span>
)
<button class="copy-code-btn"></button></code></pre>
<p><strong>Resaltado de resultados</strong></p>
<p>Cuando se habilita el resaltado de términos de consulta para el filtrado, Milvus devuelve el texto resaltado en un campo dedicado de <code translate="no">highlight</code>. Por defecto, la salida resaltada se devuelve como un fragmento a partir del primer término coincidente.</p>
<p>En este ejemplo, el primer término coincidente es <code translate="no">&quot;text&quot;</code>, por lo que el texto resaltado devuelto comienza a partir de esa posición:</p>
<pre><code translate="no" class="language-json"><span class="hljs-punctuation">{</span>

<pre><code translate="no" class="language-python"><span class="hljs-keyword">from</span> pymilvus <span class="hljs-keyword">import</span> LexicalHighlighter

highlighter = LexicalHighlighter(

<ul>
<li><p><code translate="no">fragment_offset</code> reserva el contexto inicial antes del primer término resaltado.</p></li>
<li><p><code translate="no">fragment_size</code> limita la cantidad de texto que se incluye en cada fragmento.</p></li>
<li><p><code translate="no">num_of_fragments</code> controla cuántos fragmentos se devuelven.</p></li>
</ul>
