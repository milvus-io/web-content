<p>Supongamos que tenemos dos instancias Milvus, <code translate="no">milvus_A</code> y <code translate="no">milvus_B</code>, ambas utilizando el motor de almacenamiento predeterminado MinIO para el almacenamiento de objetos. Estas instancias utilizan diferentes buckets bucket_A y <code translate="no">bucket_B</code> dentro del mismo almacenamiento de objetos. En este ejemplo, nuestro objetivo es completar las siguientes tareas:</p>
<ol>
<li><p>Crear una copia de seguridad (<code translate="no">my_backup</code>) para la colección <code translate="no">coll</code> en <code translate="no">bucket_A</code> y almacenar la copia de seguridad en <code translate="no">bucket_B</code>.</p></li>
<li><p>En <code translate="no">bucket_B</code>, restaurar desde la copia de seguridad y nombrar la colección restaurada <code translate="no">coll_bak</code>.</p></li>
</ol>

    </button></h2><h3 id="Step-1-Prepare-configuration" class="common-anchor-header">Paso 1: Preparar la configuración</h3><p>Vaya al directorio del proyecto milvus-backup y cree un directorio llamado configs:</p>
<pre><code translate="no" class="language-shell">mkdir configs
cd configs
<button class="copy-code-btn"></button></code></pre>
<p>Descargue el archivo config de copia de seguridad <code translate="no">backup.yaml</code>:</p>
<pre><code translate="no" class="language-shell">wget https://raw.githubusercontent.com/zilliztech/milvus-backup/main/configs/backup.yaml
<button class="copy-code-btn"></button></code></pre>
<p>La estructura del archivo es la siguiente</p>
<pre><code translate="no">├── configs
│   └── backup.yaml
├── milvus-backup
└── README.md
<button class="copy-code-btn"></button></code></pre>
<h3 id="Step-2-Edit-configuration-file" class="common-anchor-header">Paso 2: Editar el archivo de configuración</h3><p>Modifique el archivo backup.yaml para establecer las configuraciones adecuadas para<code translate="no">milvus_A</code>:</p>
<ul>
<li><p>Conexión configs</p>
<pre><code translate="no" class="language-yaml"><span class="hljs-comment"># milvus proxy address, compatible to milvus.yaml</span>

  <span class="hljs-attr">port:</span> <span class="hljs-number">19530</span>

<li><p><code translate="no">milvus.port</code>: Puerto TCP en el que escucha el servidor Milvus (por defecto 19530).</p></li>

<li><p>Configuración del almacenamiento (ajustes MinIO/S3)</p>

<li><p><code translate="no">minio.bucketName</code>: Nombre del bucket utilizado para el almacenamiento de datos en <code translate="no">milvus_A</code>. En este ejemplo, se establece en <code translate="no">bucket_A</code>.</p></li>
<li><p><code translate="no">minio.rootPath</code>: Ruta raíz dentro del cubo donde se almacenan los datos de <code translate="no">milvus_A</code>. En este ejemplo, se establece en <code translate="no">files</code>.</p></li>
<li><p><code translate="no">minio.backupBucketName</code>: Nombre del bucket utilizado para el almacenamiento de copias de seguridad en<code translate="no">milvus_B</code>. En este ejemplo, <code translate="no">milvus_A</code> y <code translate="no">milvus_B</code> utilizan cubos diferentes. Por lo tanto, establezca <code translate="no">bucket_B</code>.</p></li>
<li><p><code translate="no">minio.backupRootPath</code>: Ruta raíz dentro del cubo designado para almacenar los archivos de copia de seguridad en <code translate="no">milvus_B</code>. En este ejemplo, defínala como <code translate="no">backup</code>.</p></li>
</ul>
<h3 id="Step-3-Create-backup" class="common-anchor-header">Paso 3: Creación de la copia de seguridad</h3><p>Una vez guardado backup.yaml, cree una copia de seguridad denominada <code translate="no">my_backup</code>:</p>
<pre><code translate="no" class="language-shell">./milvus-backup create -c coll -n my_backup
<button class="copy-code-btn"></button></code></pre>
<p>Este comando crea la copia de seguridad <code translate="no">bucket_B/backup/my_backup</code> en el almacenamiento de objetos para la colección coll.</p>

<h3 id="Step-2-Restore-backup" class="common-anchor-header">Paso 2: Restaurar la copia de seguridad</h3><p>Restaure la copia de seguridad en <code translate="no">milvus_B</code>:</p>
<pre><code translate="no" class="language-shell">./milvus-backup restore -c coll -n my_backup -s _bak
<button class="copy-code-btn"></button></code></pre>
<p>Este comando restaura la copia de seguridad en una nueva colección denominada <code translate="no">coll_bak</code> en <code translate="no">milvus_B</code>, con los datos almacenados en <code translate="no">bucket_B/files/insert_log/[ID of new collection]</code>.</p>
