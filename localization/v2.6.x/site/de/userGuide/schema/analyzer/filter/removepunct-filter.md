  Der removepunct-Filter entfernt Interpunktionszeichen, Leerzeichen und
  Zeilenumbrüche, die einige Tokenizer - wie jieba, lindera und icu -
  normalerweise beibehalten. Verwenden Sie ihn, wenn Sie einen sauberen
  Token-Stream wünschen, der nur sinnvolle Text-Token enthält, frei von Kommas,
  Punkten und anderen Satzzeichen.

    </button></h1><p>Der Filter <code translate="no">removepunct</code> entfernt eigenständige Interpunktionszeichen aus dem Tokenstrom. Verwenden Sie ihn, wenn Sie eine saubere Textverarbeitung wünschen, die sich auf sinnvolle Inhaltswörter und nicht auf Satzzeichen konzentriert.</p>
<div class="alert note">
<p>Dieser Filter ist am effektivsten mit den Tokenizern <code translate="no">jieba</code>, <code translate="no">lindera</code> und <code translate="no">icu</code>, die Satzzeichen als separate Token erhalten (z.B. <code translate="no">&quot;Hello!&quot;</code> → <code translate="no">[&quot;Hello&quot;, &quot;!&quot;]</code>). Andere Tokenizer wie <code translate="no">standard</code> und <code translate="no">whitespace</code> lassen die Interpunktion bei der Tokenisierung weg, so dass <code translate="no">removepunct</code> keine Auswirkungen auf sie hat.</p>
</div>

<p>Der Filter <code translate="no">removepunct</code> arbeitet mit den Begriffen, die vom Tokenizer erzeugt wurden, und muss daher in Kombination mit einem Tokenizer verwendet werden.</p>
<p>Nachdem Sie <code translate="no">analyzer_params</code> definiert haben, können Sie sie bei der Definition eines Sammelschemas auf ein <code translate="no">VARCHAR</code> Feld anwenden. Dadurch kann Milvus den Text in diesem Feld unter Verwendung des angegebenen Analysators für eine effiziente Tokenisierung und Filterung verarbeiten. Einzelheiten finden Sie unter <a href="/docs/de/analyzer-overview.md#Example-use">Beispielanwendung</a>.</p>
