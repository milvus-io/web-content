<div class="alert note" id="standalone-deployment-note">
<p><strong>Benötigen Sie stattdessen eine eigenständige Bereitstellung?</strong></p>
<p>Wenn Sie Milvus für Entwicklungs- oder Testzwecke lieber im Standalone-Modus (einzelner Knoten) bereitstellen möchten, verwenden Sie diesen Befehl:</p>
<pre><code translate="no" class="language-bash">helm install my-release zilliztech/milvus \
  --<span class="hljs-built_in">set</span> image.all.tag=v2.6.19 \
  --<span class="hljs-built_in">set</span> cluster.enabled=<span class="hljs-literal">false</span> \
  --<span class="hljs-built_in">set</span> pulsarv3.enabled=<span class="hljs-literal">false</span> \
  --<span class="hljs-built_in">set</span> standalone.messageQueue=woodpecker \
  --<span class="hljs-built_in">set</span> woodpecker.enabled=<span class="hljs-literal">true</span> \
  --<span class="hljs-built_in">set</span> streaming.enabled=<span class="hljs-literal">true</span>
<button class="copy-code-btn"></button></code></pre>
<p><strong>Hinweis</strong>: Im Standalone-Modus wird Woodpecker standardmäßig als Nachrichtenwarteschlange verwendet und die Komponente „Streaming Node“ aktiviert. Weitere Informationen finden Sie unter <a href="/docs/de/v2.6.x/architecture_overview.md">„Architekturübersicht</a> “ und <a href="/docs/de/v2.6.x/use-woodpecker.md">„Woodpecker verwenden</a>“.</p>
</div>
<p><strong>Milvus-Cluster bereitstellen:</strong></p>
<p>Der folgende Befehl stellt einen Milvus-Cluster mit optimierten Einstellungen für v2.6.19 bereit und verwendet dabei Woodpecker als empfohlene Nachrichtenwarteschlange:</p>
<pre><code translate="no" class="language-bash">helm install my-release zilliztech/milvus \
  --<span class="hljs-built_in">set</span> image.all.tag=v2.6.19 \
  --<span class="hljs-built_in">set</span> pulsarv3.enabled=<span class="hljs-literal">false</span> \
  --<span class="hljs-built_in">set</span> woodpecker.enabled=<span class="hljs-literal">true</span> \
  --<span class="hljs-built_in">set</span> streaming.enabled=<span class="hljs-literal">true</span> \
  --<span class="hljs-built_in">set</span> indexNode.enabled=<span class="hljs-literal">false</span>
<button class="copy-code-btn"></button></code></pre>
<p><strong>Funktionsweise dieses Befehls:</strong></p>
<ul>
<li>Verwendet <strong>Woodpecker</strong> als Nachrichtenwarteschlange (empfohlen für geringeren Wartungsaufwand)</li>
<li>Aktiviert die neue <strong>„Streaming</strong> Node“-Komponente für eine verbesserte Leistung</li>
<li>Deaktiviert den veralteten <strong>Index-Knoten</strong> (die Funktionalität wird nun vom Datenknoten übernommen)</li>
<li>Deaktiviert Pulsar, um stattdessen Woodpecker zu verwenden</li>
</ul>
<div class="alert note">
<p><strong>Architekturänderungen in Milvus 2.6.x:</strong></p>
<ul>
<li><strong>Nachrichtenwarteschlange</strong>: <strong>Woodpecker</strong> wird nun empfohlen (reduziert den Wartungsaufwand für die Infrastruktur im Vergleich zu Pulsar)</li>
<li><strong>Neue Komponente</strong>: <strong>„Streaming Node“</strong> wird eingeführt und ist standardmäßig aktiviert</li>
<li><strong>Zusammengeführte Komponenten</strong>: <strong>Index-Knoten</strong> und <strong>Datenknoten</strong> werden zu einem einzigen <strong>Datenknoten</strong> zusammengefasst</li>
</ul>
<p>Ausführliche Informationen zur Architektur finden Sie in der <a href="/docs/de/v2.6.x/architecture_overview.md">Architekturübersicht</a>.</p>
</div>
<p><strong>Alternative Optionen für die Nachrichtenwarteschlange:</strong></p>
<p>Wenn Sie <strong>Pulsar</strong> (die herkömmliche Wahl) anstelle von Woodpecker verwenden möchten:</p>
<pre><code translate="no" class="language-bash">helm install my-release zilliztech/milvus \
  --<span class="hljs-built_in">set</span> image.all.tag=v2.6.19 \
  --<span class="hljs-built_in">set</span> streaming.enabled=<span class="hljs-literal">true</span> \
  --<span class="hljs-built_in">set</span> indexNode.enabled=<span class="hljs-literal">false</span>
<button class="copy-code-btn"></button></code></pre>
<p><strong>Nächste Schritte:</strong>
Der obige Befehl stellt Milvus mit den empfohlenen Konfigurationen bereit. Für den Produktiveinsatz:</p>
<ul>
<li>Verwenden Sie das <a href="https://milvus.io/tools/sizing">Milvus-Dimensionierungstool</a>, um die Einstellungen basierend auf Ihrer Datenmenge zu optimieren</li>
<li>Lesen Sie <a href="https://milvus.io/docs/system_configuration.md">die Milvus-Checkliste</a> für <a href="https://milvus.io/docs/system_configuration.md">Systemkonfigurationen</a>, um erweiterte Konfigurationsoptionen zu erfahren</li>
</ul>
<div class="alert note">
<p><strong>Wichtige Hinweise:</strong></p>
<ul>
<li><strong>Benennung der Releases</strong>: Verwenden Sie ausschließlich Buchstaben, Zahlen und Bindestriche (keine Punkte erlaubt)</li>
<li><strong>Kubernetes v1.25+</strong>: Wenn Probleme mit dem „PodDisruptionBudget“ auftreten, wenden Sie diese Problemumgehung an:
<pre><code translate="no" class="language-bash">helm install my-release zilliztech/milvus \
  --<span class="hljs-built_in">set</span> pulsar.bookkeeper.pdb.usePolicy=<span class="hljs-literal">false</span> \
  --<span class="hljs-built_in">set</span> pulsar.broker.pdb.usePolicy=<span class="hljs-literal">false</span> \
  --<span class="hljs-built_in">set</span> pulsar.proxy.pdb.usePolicy=<span class="hljs-literal">false</span> \
  --<span class="hljs-built_in">set</span> pulsar.zookeeper.pdb.usePolicy=<span class="hljs-literal">false</span>
<button class="copy-code-btn"></button></code></pre></li>
</ul>
<p>Weitere Informationen finden Sie <a href="https://artifacthub.io/packages/helm/milvus/milvus">im Milvus-Helm-Chart</a> und in <a href="https://helm.sh/docs/">der Helm-Dokumentation</a>.</p>
</div>

</ul>
</div>
