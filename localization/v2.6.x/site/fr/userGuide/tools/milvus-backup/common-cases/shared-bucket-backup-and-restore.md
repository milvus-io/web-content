<p>Supposons que nous ayons des instances Milvus, <code translate="no">milvus_A</code> et <code translate="no">milvus_B</code>, qui utilisent toutes deux le moteur de stockage MinIO par défaut pour le stockage d'objets. Ces instances partagent le même seau, <code translate="no">bucket_A</code>, mais stockent leurs données dans des chemins racine différents : <code translate="no">files_A</code> pour <code translate="no">milvus_A</code> et files_B pour <code translate="no">milvus_B</code>. Dans cet exemple, notre objectif est de réaliser les tâches suivantes :</p>
<ol>
<li><p>Créer une sauvegarde (my_backup) pour la collection coll qui est stockée sous le chemin<code translate="no">files_A</code> pour <code translate="no">milvus_A</code>.</p></li>
<li><p>Restaurer la sauvegarde et la stocker dans files_B pour <code translate="no">milvus_B</code>.</p></li>
</ol>

<pre><code translate="no" class="language-shell">wget https://raw.githubusercontent.com/zilliztech/milvus-backup/main/configs/backup.yaml
<button class="copy-code-btn"></button></code></pre>
<p>La structure du fichier ressemble à ceci :</p>
<pre><code translate="no">├── configs

<ul>
<li><p><code translate="no">minio.bucketName</code>: Nom du seau utilisé pour le stockage sur <code translate="no">milvus_A</code>. Dans cet exemple, il s'agit de <code translate="no">bucket_A</code>.</p></li>
<li><p><code translate="no">minio.rootPath</code>: Chemin racine dans le seau où les données de <code translate="no">milvus_A</code> sont stockées. Dans cet exemple, il s'agit de <code translate="no">files_A</code>.</p></li>
<li><p><code translate="no">minio.backupBucketName</code>: Nom du panier utilisé pour le stockage. Dans cet exemple, <code translate="no">milvus_A</code> et <code translate="no">milvus_B</code> partagent le même panier. Par conséquent, la valeur est<code translate="no">bucket_A</code>.</p></li>
<li><p><code translate="no">minio.backupRootPath</code>: Chemin d'accès racine dans le seau désigné pour le stockage des fichiers de sauvegarde dans <code translate="no">milvus_B</code>. </ul></li>
</ul>
<h3 id="Step-3-Create-backup" class="common-anchor-header">Étape 3 : Créer une sauvegarde</h3><p>Une fois que <code translate="no">backup.yaml</code> est sauvegardé, créez une sauvegarde nommée my_backup :</p>
<pre><code translate="no" class="language-shell">./milvus-backup create -c coll -n my_backup
<button class="copy-code-btn"></button></code></pre>
<p>Cette commande crée la sauvegarde <code translate="no">bucket_A/backup/my_backup</code> dans le stockage d'objets pour la collection <code translate="no">coll</code>.</p>

<h3 id="Step-2-Restore-backup" class="common-anchor-header">Étape 2 : Restauration de la sauvegarde</h3><p>Restaurer la sauvegarde sur <code translate="no">milvus_B</code>:</p>
<pre><code translate="no" class="language-shell">./milvus-backup restore -c coll -n my_backup -s _bak
<button class="copy-code-btn"></button></code></pre>
<p>Cette commande restaure la sauvegarde dans une nouvelle collection nommée <code translate="no">coll_bak</code> dans <code translate="no">milvus_B</code>, avec des données stockées dans <code translate="no">bucket_A/files_B/insert_log/[ID of new collection]</code>.</p>
