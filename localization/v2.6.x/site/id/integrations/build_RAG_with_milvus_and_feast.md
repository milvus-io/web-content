<p>Dalam tutorial ini, kita akan membangun pipeline Retrieval-Augmented Generation (RAG) menggunakan <a href="https://github.com/feast-dev/feast">Feast</a> dan <a href="https://milvus.io/">Milvus</a>. Feast adalah penyimpanan fitur sumber terbuka yang menyederhanakan manajemen fitur untuk pembelajaran mesin, memungkinkan penyimpanan dan pengambilan data terstruktur yang efisien untuk pelatihan dan inferensi waktu nyata. Milvus adalah basis data vektor berkinerja tinggi yang dirancang untuk pencarian kemiripan yang cepat, sehingga ideal untuk mengambil dokumen yang relevan dalam alur kerja RAG.</p>
<p>Pada dasarnya, kita akan menggunakan Feast untuk menyuntikkan dokumen dan data terstruktur (yaitu fitur) ke dalam konteks LLM (Large Language Model) untuk mendukung Aplikasi RAG (Retrieval Augmented Generation) dengan Milvus sebagai basis data vektor online.</p>

    </button></h1><p>Feast memecahkan beberapa masalah umum dalam aliran ini:</p>
<ol>
<li><strong>Pengambilan secara online:</strong> Pada saat inferensi, LLM sering kali membutuhkan akses ke data yang tidak tersedia dan perlu dihitung terlebih dahulu dari sumber data lain.<ul>
<li>Feast mengelola penerapan ke berbagai toko online (misalnya Milvus, DynamoDB, Redis, Google Cloud Datastore) dan memastikan fitur-fitur yang diperlukan <em>tersedia</em> secara konsisten dan <em>baru dikomputasi</em> pada waktu inferensi.</li>
</ul></li>
<li><strong>Pencarian Vektor:</strong> Feast telah membangun dukungan untuk pencarian kemiripan vektor yang mudah dikonfigurasi secara deklaratif sehingga pengguna dapat fokus pada aplikasi mereka. Milvus menyediakan kemampuan pencarian kemiripan vektor yang kuat dan efisien.</li>
<li><strong>Data terstruktur yang lebih kaya:</strong> Bersamaan dengan pencarian vektor, pengguna dapat meminta bidang terstruktur standar untuk dimasukkan ke dalam konteks LLM untuk pengalaman pengguna yang lebih baik.</li>
<li><strong>Fitur/Konteks dan pembuatan versi:</strong> Tim yang berbeda dalam sebuah organisasi sering kali tidak dapat menggunakan kembali data di seluruh proyek dan layanan, yang menghasilkan logika aplikasi ganda. Model memiliki ketergantungan data yang perlu diversi, misalnya ketika menjalankan tes A/B pada versi model/prompt.<ul>
<li>Feast memungkinkan penemuan dan kolaborasi pada dokumen, fitur, dan memungkinkan pembuatan versi dari kumpulan data.</li>
</ul></li>
</ol>
<p>Kami akan melakukannya:</p>
<ol>
<li>Menerapkan toko fitur lokal dengan <strong>toko offline file Parquet</strong> dan <strong>toko online Milvus</strong>.</li>
<li>Menulis/mewujudkan data (yaitu, nilai fitur) dari toko offline (file parket) ke toko online (Milvus).</li>
<li>Sajikan fitur-fitur menggunakan SDK Feast dengan kemampuan pencarian vektor Milvus</li>
<li>Menyuntikkan dokumen ke dalam konteks LLM untuk menjawab pertanyaan</li>
</ol>

<pre><code translate="no" class="language-python"><span class="hljs-keyword">import</span> torch
<span class="hljs-keyword">import</span> torch.nn.functional <span class="hljs-keyword">as</span> F

<span class="hljs-keyword">def</span> <span class="hljs-title function_">mean_pooling</span>(<span class="hljs-params">model_output, attention_mask</span>):
    token_embeddings = model_output[
        <span class="hljs-number">0</span>
    ]  <span class="hljs-comment"># First element of model_output contains all token embeddings</span>
    input_mask_expanded = (
        attention_mask.unsqueeze(-<span class="hljs-number">1</span>).expand(token_embeddings.size()).<span class="hljs-built_in">float</span>()
    )
    <span class="hljs-keyword">return</span> torch.<span class="hljs-built_in">sum</span>(token_embeddings * input_mask_expanded, <span class="hljs-number">1</span>) / torch.clamp(
        input_mask_expanded.<span class="hljs-built_in">sum</span>(<span class="hljs-number">1</span>), <span class="hljs-built_in">min</span>=<span class="hljs-number">1e-9</span>
    )


<span class="hljs-keyword">def</span> <span class="hljs-title function_">run_model</span>(<span class="hljs-params">sentences, tokenizer, model</span>):
    encoded_input = tokenizer(
        sentences, padding=<span class="hljs-literal">True</span>, truncation=<span class="hljs-literal">True</span>, return_tensors=<span class="hljs-string">&quot;pt&quot;</span>
    )
    <span class="hljs-comment"># Compute token embeddings</span>
    <span class="hljs-keyword">with</span> torch.no_grad():
        model_output = model(**encoded_input)

    sentence_embeddings = mean_pooling(model_output, encoded_input[<span class="hljs-string">&quot;attention_mask&quot;</span>])
    sentence_embeddings = F.normalize(sentence_embeddings, p=<span class="hljs-number">2</span>, dim=<span class="hljs-number">1</span>)
    <span class="hljs-keyword">return</span> sentence_embeddings
