    </button></h1><p>Panduan ini menunjukkan cara menggunakan dua cluster Milvus mandiri dengan Milvus Operator dan mengonfigurasi replikasi CDC dari cluster sumber ke cluster target.</p>
<p>Contoh-contoh menggunakan:</p>
<ul>
<li><code translate="no">source-cluster</code> sebagai cluster utama.</li>
<li><code translate="no">target-cluster</code> sebagai cluster siaga.</li>
<li><code translate="no">milvus</code> sebagai ruang nama untuk cluster Milvus.</li>
<li><code translate="no">milvus-operator</code> sebagai ruang nama untuk Milvus Operator.</li>
</ul>
<p>Sebelum memulai, baca <a href="/docs/id/v2.6.x/milvus_cdc_overview.md">Milvus CDC</a> untuk memahami model primary-standby dan opsi failover.</p>

<p>Siapkan dua jenis alamat:</p>
<ul>
<li>Alamat cluster dituliskan pada konfigurasi replikasi dan digunakan oleh komponen CDC. Alamat-alamat ini harus dapat dijangkau dari pod CDC.</li>

</ul>

<span class="hljs-comment"># Keep source_cluster_addr and target_cluster_addr reachable from CDC pods.</span>

<span class="hljs-comment"># target_client_addr = &quot;http://127.0.0.1:19531&quot;</span>

source_cluster_id = <span class="hljs-string">&quot;source-cluster&quot;</span>

    <span class="hljs-string">f&quot;<span class="hljs-subst">{source_cluster_id}</span>-rootcoord-dml_<span class="hljs-subst">{i}</span>&quot;</span>

<p>Ganti alamat dengan alamat layanan Milvus yang sebenarnya di lingkungan Anda. Daftar pchannel harus sesuai dengan penerapan Milvus Anda. Jangan menyalin nilai contoh tanpa memeriksa konfigurasi cluster Anda.</p>

        ></path>
      </svg>
    </button></h2><p>CDC lag adalah jendela data antara cluster utama dan cluster siaga. Anda harus memantaunya terus menerus setelah replikasi dikonfigurasi.</p>
<p>Jeda CDC dapat meningkat ketika:</p>
<ul>
<li>Laju penulisan primer tinggi.</li>
<li>Latensi jaringan atau kehilangan paket meningkat di antara cluster.</li>
<li>Cluster siaga kelebihan beban.</li>
<li>Node CDC kekurangan pasokan.</li>
<li>Operasi DDL atau impor yang besar sedang berjalan.</li>
</ul>
<p>Gunakan jeda CDC untuk memandu keputusan operasional:</p>
<ul>
<li>Jika jeda rendah, peralihan akan selesai lebih cepat.</li>
<li>Jika jeda tinggi, peralihan mungkin kehilangan lebih banyak data.</li>
</ul>
<p>Anda dapat memperkirakan jeda CDC dengan kueri PromQL berikut ini:</p>
<pre><code translate="no" class="language-promql">clamp_min(
  max by (channel_name) (
    milvus_wal_last_confirmed_time_tick
  )
  -
  min by (channel_name) (
    milvus_cdc_last_replicated_time_tick
  ),
  0
)
</code></pre>
<p>Hasilnya dalam hitungan detik. Untuk setiap saluran sumber, kueri membandingkan timetick WAL terakhir yang dikonfirmasi dengan timetick terakhir yang direplikasi oleh CDC. Jika sebuah primer mereplikasi ke beberapa cluster siaga, ekspresi <code translate="no">min by (channel_name)</code> melaporkan kemajuan replikasi yang paling lambat untuk saluran tersebut.</p>
<p>Jika Prometheus mengikis beberapa cluster Milvus, tambahkan filter label yang sesuai dengan penerapan Anda, seperti <code translate="no">namespace</code> atau <code translate="no">app_kubernetes_io_instance</code>, untuk menghindari pencampuran metrik dari cluster yang berbeda.</p>
