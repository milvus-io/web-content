Tutti i componenti di Milvus Standalone sono racchiusi in un'unica <a href="/docs/it/install_standalone-docker.md">immagine Docker</a>, il che rende comoda la distribuzione. Questo argomento descrive come scalare un'istanza di Milvus in esecuzione in questa modalità.</p>

        ></path>
      </svg>
    </button></h2><p>Quando si distribuisce Milvus Standalone con <a href="/docs/it/install_standalone-docker.md">Docker</a> o <a href="/docs/it/install_standalone-docker-compose.md">Docker Compose</a>, lo script di distribuzione (<code translate="no">standalone_embed.sh</code>) o il file di configurazione (<code translate="no">docker-compose.yml</code>) crea diversi volumi e li mappa in directory host per garantire la persistenza dei dati.</p>
<p>Per scalare un'istanza Milvus distribuita in questo modo, è necessario arrestare e rimuovere il contenitore o lo stack di contenitori esistente, distribuire nuovamente Milvus Standalone con le impostazioni di configurazione aggiornate e riutilizzare i dati persistenti sull'host per lanciare una nuova istanza.</p>
<p>La tabella seguente elenca la mappatura dei volumi tra l'host e i container.</p>
<table>
   <tr>
     <th><p>Opzione di distribuzione</p></th>
     <th><p>Percorso dell'host</p></th>
     <th><p>Percorso del contenitore</p></th>
   </tr>
   <tr>
     <td rowspan="3"><p>Docker</p></td>
     <td><p><code translate="no">$(pwd)/volumes/milvus</code></p></td>
     <td><p><code translate="no">/var/lib/milvus</code></p></td>
   </tr>
   <tr>
     <td><p><code translate="no">$(pwd)/embedEtcd.yaml</code></p></td>
     <td><p><code translate="no">/milvus/configs/embedEtcd.yaml</code></p></td>
   </tr>
   <tr>
     <td><p><code translate="no">$(pwd)/user.yaml</code></p></td>
     <td><p><code translate="no">/milvus/configs/user.yaml</code></p></td>
   </tr>
   <tr>
     <td rowspan="3"><p>Docker Compose</p></td>
     <td><p><code translate="no">${DOCKER_VOLUME_DIRECTORY:-.}/volumes/etcd</code>(milvus-etcd)</p></td>
     <td><p><code translate="no">/etcd</code></p></td>
   </tr>
   <tr>
     <td><p><code translate="no">${DOCKER_VOLUME_DIRECTORY:-.}/volumes/minio</code>(milvus-minio)</p></td>
     <td><p><code translate="no">/minio_data</code></p></td>
   </tr>
   <tr>
     <td><p><code translate="no">${DOCKER_VOLUME_DIRECTORY:-.}/volumes/milvus</code>(milvus-standalone)</p></td>
     <td><p><code translate="no">/var/lib/milvus</code></p></td>
   </tr>
</table>
<p>Prima di eseguire le procedure di questa guida, assicuratevi che i vostri dati siano presenti nei percorsi degli host di cui sopra.</p>

    </button></h2><p>Per scalare un'istanza Milvus in esecuzione, è necessario arrestare l'istanza, rimuovere il contenitore e distribuire nuovamente l'istanza con nuove impostazioni e dati persistenti.</p>
<p>La procedura specifica è la seguente:</p>
<ol>
<li><p>Eseguire <code translate="no">docker stats milvus-standalone</code> per visualizzare la CPU e la memoria allocate all'istanza Milvus. L'output dovrebbe essere simile al seguente:</p>
<pre><code translate="no" class="language-bash">CONTAINER ID   NAME                CPU %     MEM USAGE / LIMIT     MEM %     NET I/O       BLOCK I/O         PIDS

<button class="copy-code-btn"></button></code></pre>
<p>Nell'output del comando, è possibile trovare l'utilizzo attuale delle risorse dell'istanza Milvus.</p></li>
<li><p>Fermare e rimuovere il contenitore.</p>
<pre><code translate="no" class="language-bash">$ docker stop milvus-standalone
$ docker <span class="hljs-built_in">rm</span> milvus-standalone
<button class="copy-code-btn"></button></code></pre></li>
<li><p>Individuare il file di script <code translate="no">standalone_embed.sh</code>, trovare il comando <code translate="no">docker run</code> e aggiungere i limiti delle risorse.</p>
