# Arabic Normalization

The `arabic_normalization` filter is a built-in token filter for Arabic text. It normalizes Arabic-specific letter variants and removes optional marks that can make equivalent Arabic terms appear different during text analysis.

## Configuration

<div class="alert note">

For Arabic text, use the built-in [`arabic`](arabic-analyzer.md) analyzer in most cases. The built-in analyzer includes this filter together with standard tokenization, lowercasing, decimal digit normalization, Arabic stemming, and Arabic stop-word removal. Use `arabic_normalization` directly only when you need to build a custom analyzer pipeline.

</div>

To use the `arabic_normalization` filter in a custom analyzer, add it to the `filter` section in `analyzer_params`:

```python
analyzer_params = {
    "tokenizer": "standard",
    "filter": ["arabic_normalization"],
}
```

The `arabic_normalization` filter has no configurable parameters.

The filter applies the following transformations:

<table>
   <tr>
     <th><p>Transformation</p></th>
     <th><p>From</p></th>
     <th><p>To</p></th>
   </tr>
   <tr>
     <td><p>Hamza + Alef variants</p></td>
     <td><p><code>آ</code>, <code>أ</code>, <code>إ</code></p></td>
     <td><p><code>ا</code></p></td>
   </tr>
   <tr>
     <td><p>Teh Marbuta</p></td>
     <td><p><code>ة</code></p></td>
     <td><p><code>ه</code></p></td>
   </tr>
   <tr>
     <td><p>Alef Maksura</p></td>
     <td><p><code>ى</code></p></td>
     <td><p><code>ي</code></p></td>
   </tr>
   <tr>
     <td><p>Harakat</p></td>
     <td><p><code>U+064B</code> through <code>U+065F</code></p></td>
     <td><p>Removed</p></td>
   </tr>
   <tr>
     <td><p>Tatweel / Kashida</p></td>
     <td><p><code>ـ</code></p></td>
     <td><p>Removed</p></td>
   </tr>
</table>

The filter operates on tokens generated by the tokenizer. The configuration above is intentionally a custom analyzer example and does not include the complete Arabic processing pipeline.

## Examples

Before applying the analyzer configuration to your collection schema, verify its behavior using the `run_analyzer` method.

### Analyzer configuration

```python
analyzer_params = {
    "tokenizer": "standard",
    "filter": ["arabic_normalization"],
}
```

### Verification using `run_analyzer`

```python
from pymilvus import MilvusClient

client = MilvusClient(uri="http://localhost:19530")

sample_text = "آدم أحمد إسلام مدرسة كبرى كِتَابٌ عـــربي"

result = client.run_analyzer(sample_text, analyzer_params)
print(result)
```

### Expected output

```
['ادم', 'احمد', 'اسلام', 'مدرسه', 'كبري', 'كتاب', 'عربي']
```
